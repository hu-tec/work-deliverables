Codex 매뉴얼

Windows 사용자가 WSL에 접속해 Codex CLI를 설치하고 실행하는 과정을 한 화면에서 확인하는 HTML 인덱스입니다.

한눈에 보는 진행 순서

1. WSL 준비

PowerShell 관리자 권한에서 wsl --install을 실행하고 Ubuntu 계정을 만듭니다.

2. 기본 도구 설치

Ubuntu에서 기본 Linux 명령어, Node.js, npm 설치 상태를 확인합니다.

3. Codex 설치

Codex CLI를 설치하고 로그인 또는 API 키 방식으로 인증합니다.

4. 작업 폴더 실행

프로젝트 폴더로 이동한 뒤 codex를 실행해 작업을 시작합니다.

공식 기준 메모

Codex 공식 매뉴얼 기준으로 CLI는 --cd/-C, --sandbox/-s, --ask-for-approval/-a 같은 실행 옵션을 제공합니다. 로컬 작업에는 보통 workspace-write와 승인 정책을 조합해 현재 작업 폴더 중심으로 권한을 제한합니다.

처음 실행할 때 따라 할 순서

  1. 1. PowerShell을 관리자 권한으로 열기

    Windows 시작 메뉴에서 PowerShell 또는 Terminal을 검색한 뒤 마우스 오른쪽 버튼으로 관리자 권한으로 실행을 선택합니다.

  2. 2. WSL과 Ubuntu 설치

    아래 명령을 실행하고, 재부팅 안내가 나오면 Windows를 다시 시작합니다.

    wsl --install

    Ubuntu를 명확히 지정하려면 wsl --install -d Ubuntu를 사용합니다.

  3. 3. Ubuntu 계정 만들기

    Ubuntu를 처음 열면 Linux 사용자 이름과 비밀번호를 만듭니다. 비밀번호 입력 중 화면에 글자가 보이지 않는 것은 정상입니다.

  4. 4. WSL 접속 확인

    PowerShell에서 설치 상태를 확인합니다.

    wsl -l -v

    VERSION이 2이면 WSL2로 실행 중입니다.

  5. 5. 작업 폴더 만들기

    Ubuntu 터미널에서 Codex로 작업할 폴더를 만듭니다.

    cd ~
mkdir -p projects/my-codex-test
cd projects/my-codex-test
pwd

  6. 6. Node.js와 npm 확인

    Codex CLI 설치에 필요한 기본 도구가 있는지 확인합니다.

    node -v
npm -v

    명령을 찾을 수 없다고 나오면 sudo apt update 후 sudo apt install -y nodejs npm을 실행합니다.

  7. 7. Codex CLI 설치

    Ubuntu 터미널에서 Codex CLI를 설치하고 버전을 확인합니다.

    npm install -g @openai/codex
codex --version

  8. 8. Codex 실행 및 로그인

    작업 폴더에서 Codex를 실행합니다. 로그인이 필요하면 터미널 안내에 따라 브라우저 로그인 또는 인증 코드를 진행합니다.

    cd ~/projects/my-codex-test
codex

초보자 주의사항

  • Codex를 실행하기 전에는 항상 pwd로 현재 위치를 확인합니다.
  • 중요한 프로젝트는 Git 커밋 또는 백업 후 작업합니다.
  • API 키는 비밀번호처럼 취급하고 화면 공유나 문서에 노출하지 않습니다.
  • Windows 바탕화면 경로는 WSL에서 /mnt/c/Users/사용자명/Desktop 형태로 보입니다.
  • 개발 프로젝트는 가능하면 ~/projects 아래에 두면 WSL에서 더 안정적으로 작업할 수 있습니다.
