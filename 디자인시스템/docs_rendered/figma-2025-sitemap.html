2. 사용자 관점 서비스 설명

일반 사용자가 보는 2025 자료는 "시험 플랫폼"에 바로 붙일 확정 기능이라기보다, 시험 플랫폼의 화면 경험을 확장할 때 참고할 수 있는 AI 작업 도구형 UI 예시를 포함한다.

창작작업실01.png는 사용자가 문서, 유튜브, 리서치, 프로그래밍, 이미지/디자인, 음악, 음성 같은 작업 유형을 고르고 파일이나 텍스트를 입력하는 방식을 보여준다. 다만 현재 해석에서는 이 화면을 독립 구현 대상이라기보다, 시험 플랫폼의 향후 UX/UI를 설계할 때 참고할 수 있는 패턴으로 본다.

관리자는 사용자, 시험, 자격증, 카테고리, 양식, 템플릿을 관리한다. 2025에서 새로 강조되는 부분은 기존 시험 관리뿐 아니라 AI 작업에 쓰이는 양식과 템플릿을 운영자가 관리한다는 점이다.

전문가는 프롬프트 출제, 단답형 출제, 채점, 정산, 전문가 프로필 관리를 수행한다. 일반 사용자 화면과 달리 전문가 영역은 문제 작성, 평가요소 설정, 답변 예시 입력처럼 시험/검수 업무 중심이다.

4.2 메타-T 번역

대표 파일: 메타번역.png

메타-T 번역은 창작작업실 참고 화면과 비슷한 레이아웃을 쓰지만, 번역 작업에 특화된 화면이다. 이 화면도 시험 플랫폼 본류 기능으로 바로 확정하기보다, 번역/편집 UX 참고 자료로 우선 분류한다.

주요 기능:

영역기능
번역 유형 탭동시통역, 문서번역, YouTube, 기타, 웹툰번역 중 선택한다.
번역 목적 칩아나운서, 관광가이드, 큐레이터, 안내 같은 세부 작업 유형을 선택한다.
카테고리 선택법률, 국내 법률, 고소장 같은 분야를 지정한다.
언어 설정출발어와 도착어를 선택하고, 영어/중국어/일어/아랍어/베트남어 등 언어 옵션을 고른다.
사용 AI번역에 사용할 AI 엔진을 선택한다.
양식 에디터결과를 편집기에서 다룰지 선택한다.
후면 작업 요청검수 요청, 전문가 요청, 긴급 여부를 지정한다.
파일/텍스트 입력번역할 파일 또는 텍스트를 입력한다.
관련 서식 추천번역 작업에 맞는 관련 서식을 제안한다.

4.4 장바구니와 결제

대표 파일: 장바구니.png, 결제.png

장바구니는 AI 작업 또는 번역 작업을 상품처럼 담아 결제 전 확인하는 화면이다.

장바구니 기능:

기능설명
전체 선택여러 작업을 한 번에 선택한다.
선택 삭제선택한 작업을 장바구니에서 제거한다.
작업 카드작업명, 첨부 파일명, 언어쌍, 번역기, 문서 변환 유형, 에디팅 사용 여부, 글자 수, 가격을 표시한다.
선택 요약선택된 번역 수, 작업명, 단가, 추가 옵션 금액을 보여준다.
총액 계산선택한 작업의 최종 결제 금액을 계산한다.
신용카드/페이 결제일반 결제 수단으로 결제한다.
포인트 결제보유 포인트를 사용한다.
구독 신청정기 구독 상품으로 전환한다.
전문가 견적 요청고급 전문가 작업이 필요한 경우 별도 견적을 요청한다.

결제 화면은 장바구니의 축약 버전이다. 선택된 작업 1건의 상세와 총 결제 금액, 결제 버튼을 보여준다.

5. 전문가 영역 상세 기능

5.1 프롬프트 출제하기

대표 파일: 상세_프롬프트출제_설정.png, 상세_프롬프트출제_확인.png, 출제관리_프롬프트상세.pdf

전문가가 프롬프트형 시험 문제를 작성하는 화면이다.

주요 기능:

영역기능
문제 탭문제1, 문제2, 문제3 등 문항 단위 이동
시험 문제수험생에게 제시할 문제 내용을 입력한다.
단계 구성1단계, 2단계, 3단계처럼 풀이 또는 작성 단계를 나눈다.
입력 유형단일선택, 복수선택, 단답형, 사용자 입력 영역 등을 지정한다.
보기 입력선택형 문항의 보기를 입력한다.
평가 설정1차 평가 요소, 2차 평가 요소, 단어수, 문장부호수, 전체 문장수, 유사어를 설정한다.

9. 사용자 플로우

9.1 창작작업실 참고 플로우

  1. 사용자가 작업형 화면에 진입한다고 가정한다.
  2. 작업 유형을 선택한다. 예: Youtube, 문서, 리서치&분석.
  3. 카테고리를 선택한다. 예: 법률 > 국내 법률 > 고소장.
  4. 사용할 AI를 선택한다.
  5. 파일 또는 텍스트를 입력한다.
  6. 필요하면 검수 요청, 고급전문가, 긴급 작업을 선택한다.
  7. 추천 서식을 선택하거나 직접 입력한다.
  8. 임시저장, 장바구니 담기, 적용 중 하나를 선택한다.
  9. 결과 편집이 필요한 경우 번역/편집 작업 화면으로 이동한다.
  10. 작업 완료 후 장바구니 또는 결제로 이동한다.

9.2 메타-T 번역 플로우

  1. 사용자가 메타-T 번역에 진입한다.
  2. 번역 유형을 선택한다. 예: 문서번역.
  3. 번역 목적을 선택한다. 예: 큐레이터.
  4. 출발어와 도착어를 선택한다.
  5. 사용할 AI 번역기를 선택한다.
  6. 파일 또는 텍스트를 입력한다.
  7. AI 번역 결과와 비교 번역 결과를 확인한다.
  8. 에디터에서 최종 번역문을 수정한다.
  9. 저장, 결제, 적용을 수행한다.

9.3 양식관리와 작업형 참고 화면 연결 플로우

  1. 관리자가 카테고리와 양식을 등록한다.
  2. 관리자가 양식에 사용할 AI, 입력 필드, 추천 서식, 검수 옵션을 설정한다.
  3. 사용자가 작업형 화면에서 해당 카테고리나 양식을 선택한다고 가정한다.
  4. 좌측 설정 패널과 우측 입력 영역이 관리자 설정값에 맞춰 표시되는 구조를 참고한다.
  5. 사용자가 입력한 작업은 AI 처리, 전문가 검수, 번역/편집, 결제 흐름으로 이어진다.

이 플로우가 Group 758531575.jpg에서 연결선으로 설명된 핵심 내용이다.

6장바구니/결제유료 작업, 포인트, 구독, 전문가 견적과 연결된다.
7전문가 출제 화면기존 시험 플랫폼 업무와 연결되는 전문 영역이다.
8관리자 회원/시험/자격증기존 IA와 겹치는 운영 기능이므로 공통 패턴을 재사용한다.

11. 확인이 필요한 미정 항목

항목확인 필요 내용
창작작업실의 성격실제 구현 대상인지, UX/UI 참고 자료인지 확정 필요. 현재 해석은 참고 자료에 가깝다.
양식과 템플릿의 차이양식은 입력 폼, 템플릿은 추천/상품화된 양식으로 보이나 정책 확정 필요
가격 계산 방식글자 수, 번역기 수, 에디팅, 긴급, 검수 옵션별 과금 규칙 확인 필요
AI 엔진별 결과 저장각 엔진 결과를 모두 저장할지 최종 결과만 저장할지 확인 필요
전문가 검수 흐름검수 요청 후 전문가 배정, 견적, 결제, 결과 전달 절차 확인 필요
파일 업로드 제한파일 형식, 용량, 개수, 보안 검사 정책 확인 필요
관리자 대시보드 지표실제 운영 KPI와 표시 우선순위 확인 필요

12. 요약

2025 구성은 "시험 플랫폼 운영 시스템" 위에 적용할 수 있는 UX/UI 참고 자료와 일부 관리자/전문가 화면을 포함한다. 특히 창작작업실과 메타-T 번역은 현재 기준에서 핵심 구현 기능이라기보다, 사용자 작업 화면과 번역/편집 화면의 참고 레이아웃으로 보는 것이 안전하다.

Group 758531575.jpg는 이 연결 구조를 설명하는 자료다. 따라서 별도 화면으로 구현하지 않고, 양식관리와 작업형 참고 화면, 번역/편집 패널 간 연결 관계를 설명하는 근거로 문서화해야 한다.

13. UX/UI 관점 보완

2025 화면은 2024의 시험 운영 화면보다 사용자가 직접 작업물을 만드는 성격이 강하다. 따라서 단순 관리자 테이블보다 “입력 → AI 처리 → 결과 확인 → 수정 → 저장/결제” 흐름이 자연스럽게 이어져야 한다.

영역UX/UI 핵심
창작작업실 참고 화면작업 유형, 카테고리, AI, 파일/텍스트 입력, 옵션을 순서대로 배치하는 UX/UI 참고 자료로 활용한다.
메타-T 번역원문, AI 번역 결과, 에디터 수정본을 비교하기 쉽게 배치해야 한다.
양식관리관리자가 만든 양식이 사용자 화면에 어떻게 보일지 미리보기로 확인할 수 있어야 한다.
템플릿 양식 관리자템플릿이 어떤 카테고리와 작업 유형에 쓰이는지 명확히 표시해야 한다.
장바구니/결제작업 옵션이 가격에 어떻게 반영되는지 사용자가 이해해야 한다.

13.1 창작작업실 참고 화면 설계 기준

화면 요소권장 설계
작업 유형 탭Youtube, 문서, 리서치&분석 등은 상단 탭으로 제공하되 선택된 탭을 명확히 표시한다.
카테고리 선택대/중/소분류가 길어질 수 있으므로 단계형 선택 또는 검색형 셀렉트를 사용한다.
AI 선택여러 AI를 동시에 비교할 수 있는지, 하나만 선택하는지 정책을 명확히 한다.
파일 입력업로드 가능 형식, 용량, 처리 상태, 실패 사유를 바로 보여준다.
옵션긴급, 전문가 검수, 고급전문가 같은 비용 발생 옵션은 가격 영향과 함께 표시한다.
실행 버튼임시저장, 장바구니, 적용의 차이를 사용자가 이해할 수 있도록 버튼 위치와 문구를 구분한다.

13.2 번역/편집 UX

항목권장 방식
원문/번역문좌우 또는 상하 분할로 비교 가능하게 배치한다.
AI별 결과ChatGPT, Gemini 등 AI별 결과를 탭이나 카드로 구분한다.
최종 편집본사용자가 수정한 최종본은 별도 저장 영역으로 분리한다.
변경 이력원문, AI 결과, 사용자 수정본의 차이를 추적할 수 있어야 한다.
저장/결제결과를 저장하는 행동과 결제로 넘어가는 행동을 분리한다.

13.3 관리자 UX

2025 관리자 화면은 기존 시험 운영과 신규 AI 서비스 설정이 섞인다. 관리자가 헷갈리지 않도록 메뉴를 다음처럼 분리하는 것이 좋다.

메뉴 그룹포함 기능

시험 운영시험관리, 자격증관리, 출제/채점 관리
사용자 운영회원관리, 전문가 프로필
판매/결제장바구니, 결제, 요금제, 포인트
콘텐츠 운영메인 화면, 추천 서식, 안내 문구
