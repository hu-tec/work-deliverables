BO 관리자용 레이아웃 기준 프롬프트

기준 파일: index.html, dense-drawer.css, table_design/table-design.css, table_design/*.html

이 문서는 BO 관리자 화면의 레이아웃 밀도와 4단 그리드 구성을 집중적으로 정의한다. 관리자 화면 전체 원칙은 03_BO_관리자화면_기준.md, 대량 테이블은 02_BO_테이블레이아웃_기준.md를 함께 적용한다.

실행 프롬프트

너는 BO 관리자 화면 레이아웃을 설계하는 시니어 UI 디자이너이자 프론트엔드 설계자다.

현재 `design_system_major` 기준을 따르되, 이번 화면은 특히 "4단 그리드", "높은 정보 밀도", "업무 패널을 한 화면에 많이 배치하는 구조"를 우선한다.

기준 파일:
- index.html
- dense-drawer.css
- table_design/table-design.css
- table_design/관리자패널_전체회원.html
- table_design/신규테이블관리.html
- table_design/결제관리.html

목표:
- 관리자 첫 화면 또는 상세 작업 화면에서 핵심 업무 정보를 4단 그리드로 압축 배치한다.
- 사용자가 스크롤을 많이 하지 않아도 상태, 요청, 처리량, 최근 이슈, 할 일, 테이블 진입점을 동시에 확인하게 한다.
- 단순히 카드 4개를 크게 나열하지 말고, 작고 조밀한 업무 패널을 4열 grid로 구성한다.

기본 App Shell:
- 좌측 Sidebar: 224px 고정 기준
- 우측 Content: `minmax(0, 1fr)`
- Topbar: 48px sticky
- Page padding: 18~20px 수준
- Section gap: 10~14px 수준
- Panel padding: 10~12px 수준
- 과한 여백을 쓰지 않는다.

4단 그리드 기본:
- 데스크톱 기준 `grid-template-columns: repeat(4, minmax(0, 1fr))`
- gap은 8~10px 기준
- 각 패널은 1열, 2열, 4열 span을 상황에 따라 사용한다.
- 기본 패널 높이는 콘텐츠에 맞추되 첫 화면이 느슨해지지 않게 한다.
- 카드 안에 또 카드를 중첩하지 않는다.

권장 CSS 구조:

.bo-layout-grid { display: grid; grid-template-columns: repeat(4, minmax(0, 1fr)); gap: 10px; align-items: start; }

.bo-panel { min-width: 0; border: 1px solid var(--line); background: var(--panel); padding: 10px 12px; }

.bo-panel--span-2 { grid-column: span 2; }

.bo-panel--span-4 { grid-column: 1 / -1; }


4단 그리드에 배치할 정보:
1. KPI 패널
   - 오늘 처리, 대기, 오류, 결제, 매칭, 문의 같은 숫자
   - 숫자는 크되 과하지 않게 14~18px 수준
   - 라벨은 8~11px 수준
   - 상태 chip을 함께 둔다.

2. 할 일 패널
   - 미배정, 환불 확인, 정산 대기, 문의 답변 같은 운영 task
   - 항목은 4~6개까지만 보이고 더보기 action을 둔다.
   - checkbox 또는 상태 chip을 사용한다.

3. 최근 활동 패널
   - 최근 로그인, 문서 업로드, 결제, 매칭 요청, 상태 변경
   - 2줄 이하 dense list로 구성한다.
   - 시간은 짧게 표기한다.

4. 알림/이슈 패널
   - 실패, 취소, 환불, 미납, 검수대기 같은 주의 항목
   - 위험도는 `status fail`, `status wait` chip으로 구분한다.

5. 빠른 액션 패널
   - 회원 추가, 엑셀 내보내기, 상태 변경, 메시지 발송, 정산 확인
   - 버튼은 작고 한 줄에 여러 개 배치한다.

6. 미니 테이블 패널
   - 최근 결제, 최근 문의, 정산 대기, 매칭 대기
   - 전체 테이블보다 작게 3~5행만 보여준다.
   - 헤더와 셀은 8px 기준이다.

7. 상세 작업 패널
   - 특정 회원/결제/매칭을 선택했을 때 기본정보, 상태변경, 메모, 이력 패널을 4단에 배치한다.
   - 상세 화면은 Drawer wide 상태에서 4단 grid로 확장할 수 있다.

권장 배치 예시:
- 1행: KPI 4개
- 2행: 할 일 1열, 알림/이슈 1열, 최근 활동 2열
- 3행: 미니 테이블 2열, 빠른 액션 1열, 운영 메모 1열
- 4행: 메인 테이블 또는 상세 작업영역 4열 span

상세 작업 화면 4단 예시:
- 1열: 회원/대상 기본정보 마스터
- 2열: 상태/권한/구독/결제 설정
- 3열: 상담/문의/메모/처리 task
- 4열: 최근 활동/결제/정산/로그
- 하단 4열 span: 관련 테이블 또는 이력 전체

밀도 기준:
- BO 기본 텍스트: 8px
- 패널 제목: 11~13px
- KPI 숫자: 14~18px
- 보조 텍스트: 8px
- 버튼 높이: 24~30px
- chip/status 높이: 18~22px
- panel padding: 10~12px
- row/list item 높이: 24~32px

패널 헤더:
- 좌측에는 제목, 우측에는 count 또는 더보기 action을 둔다.
- 제목 옆에 불필요한 긴 설명을 붙이지 않는다.
- 설명이 필요하면 작은 muted 텍스트 1줄만 사용한다.

Dense List:
- 각 항목은 `main + sub + status` 조합으로 만든다.
- 긴 문장은 말줄임 처리한다.
- 날짜는 `YY-MM-DD`, 시간은 필요한 경우 `HH:mm`만 사용한다.
- 리스트 항목 사이 border는 약하게 쓴다.

상태/유형:
- 반복 상태는 `status ok/wait/fail/info` chip을 쓴다.
- 유형, 등급, 결제수단, 우선순위는 `badge` 또는 보조 chip을 쓴다.
- 색상 텍스트만으로 상태를 표시하지 않는다.

반응형:
- 1200px 이상: 4단 유지
- 900~1199px: 2단으로 축소
- 900px 미만: 1단 또는 핵심 2단으로 축소
- 모바일에서도 BO 테이블은 카드화보다 가로 스크롤을 우선한다.

금지:
- 4단 그리드를 큰 마케팅 카드 4개로 만들지 않는다.
- 패널 안에 중첩 카드와 큰 그림을 넣지 않는다.
- 첫 화면에 빈 여백이 크게 생기게 하지 않는다.
- KPI 숫자만 있고 실제 업무 진입점이 없는 대시보드를 만들지 않는다.
- 상태를 색상 텍스트만으로 표시하지 않는다.
- 4단 그리드가 깨져 텍스트가 겹치거나 잘리게 하지 않는다.

산출물:
- Sidebar + Topbar + Page Head
- 4단 `bo-layout-grid`
- KPI 패널 4개 이상
- 할 일/알림/최근 활동/빠른 액션 패널
- 미니 테이블 또는 메인 테이블 진입 영역
- 상태/유형 chip
- 2단/1단 반응형 대응

4단 그리드 패널 구성 템플릿

위치Span패널내용
1행 1열1오늘 처리처리 수, 완료율, 상태 chip
1행 2열1대기 업무미배정, 검수대기, 정산대기
1행 3열1결제/환불결제완료, 환불요청, 미납
1행 4열1문의/알림답변대기, 긴급 문의

2행 2열1알림/이슈fail/wait 상태 목록

3행 1~2열2미니 테이블최근 결제/문의/매칭 3~5행
