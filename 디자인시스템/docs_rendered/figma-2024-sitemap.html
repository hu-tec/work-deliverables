1. 전체 서비스 개요

2024 Figma 자료는 AITP 시험 플랫폼의 전체 업무 흐름을 담고 있다. 단순한 시험 접수 사이트가 아니라, 수험생이 시험을 신청하고 응시한 뒤 결과와 자격증을 받는 흐름, 출제자가 문제를 만들고 제출하는 흐름, 채점자가 답안을 평가하는 흐름, 관리자가 회원·시험·문제·결제·운영을 통제하는 흐름이 함께 설계되어 있다.

일반 사용자가 이해하기 쉽게 말하면, 이 서비스는 다음 네 종류의 사용자를 위한 통합 시험 운영 시스템이다.

채점자배정된 답안 채점, 채점내역 확인, 피드백 작성채점관리.pdf, 채점관리_번역상세.pdf, 채점관리_프롬프트상세.pdf, 채점자 홈_시험 채점하기.png

4. 수험생 영역

수험생 영역은 “시험을 찾고 접수한다 → 결제한다 → 시험을 본다 → 결과를 확인한다 → 자격증을 신청한다”의 흐름으로 이해하면 된다.

출제 기능의 핵심 필드는 시험명, 차수, 등급, 언어쌍, 대/중/소분류, 문제 유형, 난이도, 배점, 지문, 문제, 정답, 해설, 평가기준, 첨부파일이다.

출제 프로세스는 다음과 같다.

  1. 출제자가 로그인한다.
  2. 담당 시험 또는 새 문제 생성을 선택한다.
  3. 시험/언어/분류/난이도/문제 유형을 설정한다.
  4. 지문, 문제, 정답, 해설, 평가기준을 입력한다.
  5. 미리보기로 수험생 화면 기준 표시를 확인한다.
  6. 임시저장 또는 최종제출을 한다.
  7. 관리자가 검수한다.
  8. 반려되면 수정 후 재제출하고, 승인되면 문제은행 또는 시험에 반영된다.

11. 구현 관점 우선순위

  1. 권한별 라우팅과 공통 레이아웃을 먼저 만든다.
  2. 수험생의 시험 접수/결제/응시/결과/자격증 흐름을 연결한다.
  3. 출제자 문제 작성과 관리자 문제 검수 흐름을 연결한다.
  4. 채점자 채점과 관리자 결과 확정 흐름을 연결한다.
  5. 결제/정산/환불을 시험 접수 및 자격증 신청과 연결한다.
  6. 고객지원, 게시판, FAQ, 블로그, 사이트관리, 통계를 운영관리로 묶는다.
  7. STT/TTS/Youtube/영상/번역 기능은 시험 유형 확장 모듈로 분리한다.

12. 주의 사항

현재 문서는 PDF 내부 텍스트 전문 OCR이 아니라, 파일명과 확인 가능한 대표 이미지 기준으로 정리했다. 따라서 세부 컬럼명, 버튼명, 입력값은 실제 PDF 화면을 개별 확인하면서 보완해야 한다. 다만 전체 IA와 기능 범위, 역할별 흐름은 2024 Figma 자료만으로도 명확하게 파악된다.

13. UX/UI 관점 보완

2024 화면은 기능 범위가 넓기 때문에 단순히 메뉴를 모두 노출하면 사용자가 길을 잃기 쉽다. 역할별로 “지금 해야 할 일”을 먼저 보여주고, 상세 기능은 그 다음 단계에서 펼쳐야 한다.

역할첫 화면에서 가장 먼저 보여줘야 할 것UX/UI 설계 방향
수험생접수한 시험, 오늘 응시 가능 여부, 결과 발표 여부상태 카드와 다음 행동 버튼을 크게 배치한다. 예: 결제하기, 시험시작, 결과보기, 자격증신청
출제자임시저장 문항, 반려 문항, 제출 마감이어쓰기, 수정 후 재제출, 새 문제 생성을 빠르게 누를 수 있게 한다.
채점자채점 대기 건수, 진행 중 채점, 마감 임박채점 대상 목록과 채점 시작 버튼을 우선 배치하고, 기준표는 채점 화면에서 고정 영역으로 제공한다.
관리자미처리 업무, 오류/대기 상태, 오늘의 운영 지표대시보드에서 문의 미답변, 결제 미승인, 환불 대기, 검수 대기를 바로 확인하게 한다.

13.1 목록 화면 UX

관리자와 사용자 화면 모두 목록 테이블이 많다. 테이블은 다음 기준을 적용하는 것이 좋다.

항목권장 방식
핵심 열회원명, 시험명, 상태처럼 판단에 필요한 값은 왼쪽에 고정한다.
상태값텍스트만 쓰지 말고 색상 배지와 문구를 함께 표시한다.
주요 버튼한 행에 버튼을 너무 많이 넣지 않고 가장 중요한 1개 행동만 먼저 노출한다.
가로 스크롤핵심 열은 고정하고 세부 정보는 탭 또는 상세 화면으로 분리한다.
검색 조건기본 검색과 상세 검색을 나누어 초보 관리자가 부담을 느끼지 않게 한다.
빈 목록정보 없음만 보여주지 말고 다음 행동을 안내한다. 예: 등록된 시험이 없습니다. 새 시험을 등록하세요.

13.2 입력 화면 UX

시험 등록, 출제, 채점, 문의 작성처럼 입력 항목이 많은 화면은 한 페이지에 모든 필드를 노출하면 실수가 늘어난다.

화면권장 UX
시험 등록기본정보, 일정, 접수/환불, 시험유형, 공개설정 단계로 나눈다.
문제 출제시험 선택, 문제 작성, 정답/해설, 평가기준, 미리보기, 제출 단계로 나눈다.
채점문제/답안/기준표/점수입력/코멘트를 한 화면에 두되, 기준표와 저장 버튼은 항상 보이게 한다.
결제/환불금액, 환불 가능 여부, 처리 후 결과를 명확히 보여주고 최종 확인 팝업을 둔다.
1:1 문의문의 유형, 관련 시험, 제목, 내용, 첨부파일만 우선 제공한다.

13.3 접근성과 신뢰성

항목보완 기준
색상합격/불합격, 완료/대기, 위험/취소 상태는 색상과 텍스트를 함께 사용한다.
버튼삭제, 환불, 최종제출, 성적공개 같은 되돌리기 어려운 작업은 확인 팝업과 이력 기록을 둔다.
자동저장응시, 출제, 채점 화면에는 자동저장 또는 저장 상태 표시가 필요하다.
시간 표시시험 시간, 접수 마감, 결과 발표일은 날짜와 시간을 함께 명확히 표시한다.
개인정보관리자 테이블에서는 연락처, 이메일, 주소 같은 개인정보를 권한에 따라 마스킹할 수 있어야 한다.
