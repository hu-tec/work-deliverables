## 목표

BO 사이드바를 업무 성격 기준으로 다시 묶는다.  
사이드바는 `1depth 대분류`와 `2depth 중분류`까지만 담당하고, `3depth 이하`는 콘텐츠 영역의 탭, 표, 필터, 섹션 내비게이션으로 처리한다.

현재처럼 많은 메뉴가 같은 층위에 평면 나열되면 사용자가 업무 시작점을 찾기 어렵다. 따라서 최상위 메뉴는 5개 업무 대분류와 3개 부가영역으로 정리한다.

## 정리계획

### 1단계. 현행 메뉴를 데이터로 수집

- 현재 사이드바 메뉴를 `id`, `label`, `path`, `group`, `role`, `legacy`, `tabOf` 구조로 정리한다.
- 기존 URL은 바로 삭제하지 않고 새 구조의 메뉴 또는 콘텐츠 탭으로 매핑한다.
- 중복 메뉴, 이전 메뉴, 권한 메뉴를 별도 표시한다.

### 2단계. 5개 업무 대분류로 재배치

| 1depth 대분류 | 역할 | 2depth 중분류 |
| --- | --- | --- |
| 홈 · 내 업무 | 개인 업무 시작점 | 대시보드, 업무일지, 업무 총괄, 내 출퇴근 |
| 지식 · 규정 | 내부 기준과 문서 | 사내업무지침, 규정, 매뉴얼, 서식 · 확인서, 업무 자료 |
| 인사 · 채용 | 사람, 지원자, 강사 관리 | 채용관리, 출퇴근 관리, 학생관리, 출강 · 강사관리 |
| 운영 · 거래처 | 일정, 거래처, 프로젝트 운영 | 미팅 · 일정, 강의시간표, 거래처관리, 미수금관리, 프로젝트관리, 사진모음 |
| 기획 · 콘텐츠 | 기획 자료와 외부 콘텐츠 | 업무 분류, 강사 커리큘럼, 마케팅, 프롬프트 사용법, AI 강의안, 커뮤니티 · 홈페이지 |

### 3단계. 부가영역 분리

| 영역 | 위치 | 기준 |
| --- | --- | --- |
| 즐겨찾기 | 사이드바 상단 | 사용자별 자주 쓰는 5~7개 메뉴 |
| 시스템 관리 | 사이드바 하단 | 관리자 권한이 필요한 메뉴 |
| 보관함(Legacy) | 사이드바 하단, 기본 접힘 | 이전 화면, 대체 화면, 사용 중단 후보 |

### 4단계. 3depth 이하를 콘텐츠 영역으로 이동

| 2depth 메뉴 | 콘텐츠 영역 3depth 예시 | 4depth 이하 예시 |
| --- | --- | --- |
| 규정 | 통합, 관리, 편집, 레이아웃, 매뉴얼, 평가기준 | 규정 목록, 버전, 승인 상태, 담당자 |
| 채용관리 | 지원서, 면접, 대시보드, 평가 | 지원자 목록, 면접 일정, 평가표 |
| 거래처관리 | 거래처 목록, 아웃콜, 출장 거래처, 계약 이력 | 담당자, 연락처, 미팅 기록, 미수금 |
| 출퇴근 관리 | 일별 현황, 월별 집계, 예외 처리 | 직원별 기록, 수정 요청, 승인 상태 |

### 5단계. BO 시각 기준 적용

- `border-radius: 0`을 기본값으로 한다.
- 그림자보다 1px border와 구획선을 사용한다.
- 메뉴 높이, 패딩, 목록 간격을 줄인다.
- active 상태는 좌측 bar, 배경색, 텍스트 색으로 명확히 표시한다.
- 전체 화면은 고밀도 업무 화면으로 구성한다.

## 왜 이렇게 정리하는가

### 업무 시작점이 선명해진다

BO 사용자는 메뉴를 둘러보는 것보다 업무를 빨리 시작해야 한다. `홈 · 내 업무`를 최상단에 두면 업무일지, 업무 총괄, 내 출퇴근처럼 자주 쓰는 진입점이 먼저 보인다.

### 같은 데이터의 화면이 흩어지지 않는다

규정 통합, 규정관리, 규정편집, 규정 레이아웃, 규정매뉴얼, 평가기준은 모두 규정이라는 하나의 데이터 묶음을 다룬다. 이를 사이드바 메뉴로 나누면 메뉴 수만 늘고 사용자는 어떤 메뉴가 원본인지 헷갈린다. `규정` 하나를 2depth로 두고 나머지는 콘텐츠 탭으로 두는 편이 좋다.

### 권한 경계가 드러난다

인사/채용, 시스템 관리는 일반 업무보다 민감하다. 메뉴 구조에서 따로 분리하면 권한 설계, QA, 운영 정책을 맞추기 쉽다.

### 이전 메뉴 사용 사고를 줄인다

이전 화면과 대체 화면을 현역 메뉴에 섞어두면 잘못된 화면에서 업무가 진행될 수 있다. 보관함으로 격리하면 접근은 가능하되 기본 업무 흐름에서는 빠진다.

### 유지보수가 쉬워진다

메뉴를 라우트 단위로 무작정 늘리지 않고 `업무 도메인 > 대표 화면 > 내부 탭` 구조로 두면 새 기능을 추가할 때 어느 위치에 붙일지 판단하기 쉽다.

목표:
- 사이드바는 1depth 대분류와 2depth 중분류까지만 표시한다.
- 3depth부터 하위 분류는 콘텐츠 영역에서 탭, 표, 필터, 섹션 내비게이션으로 처리한다.
- 최상위 업무 대분류는 5개 안팎으로 유지한다.
- 즐겨찾기, 시스템 관리, 보관함은 일반 업무 대분류와 분리한다.

   - 미팅 · 일정
