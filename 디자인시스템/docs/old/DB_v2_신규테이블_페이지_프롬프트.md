# DB v2 신규 테이블 별도 페이지 제작 프롬프트

아래 프롬프트는 Figma Make의 `B. DB 페이지 v2` 화면을 기준으로, 기존 관리자 화면과 분리된 신규 테이블 전용 페이지를 만들기 위한 작업 지시문이다.

## 실행 프롬프트

```text
너는 BO 관리자 화면을 설계하는 시니어 UI 디자이너이자 프론트엔드 설계자다.

아래 Figma Make 화면을 기준으로 신규 DB 테이블 전용 페이지를 새로 만들어라.
참고 화면:
https://www.figma.com/make/uGCONfLFy7YzzZEbu1yiQQ/B.-DB-%ED%8E%98%EC%9D%B4%EC%A7%80-v2?p=f&t=WYMZJlnEAs9kFsTm-0

목표:
- 기존 `B. DB 페이지 v2`의 관리자 패널 톤, 좌측 사이드바, 상단 헤더, 사이트 필터, 조밀한 테이블 밀도를 유지한다.
- 기존 페이지를 덮어쓰지 말고, 신규 테이블을 위한 별도 페이지를 만든다.
- 페이지 이름은 `DB 테이블 v3` 또는 `신규 테이블 관리`로 한다.
- 새 페이지는 "여러 신규 테이블을 빠르게 전환하고 비교하는 운영 화면"이어야 한다.

전체 레이아웃:
1. 좌측에는 기존 관리자 사이드바를 유지한다.
   - 대시보드, 회원, 활동 관리, 결제 관리, 매칭 관리, 시험 관리, 스케줄 관리, 대중소 관리, 1:1 문의, 통계 구조를 유지한다.
   - 현재 메뉴는 신규 테이블 관리 또는 DB 관리 성격으로 active 처리한다.

2. 본문 상단에는 제목 영역을 둔다.
   - 상단 breadcrumb: 관리자 > DB 테이블
   - H1: 신규 테이블 관리
   - 우측 액션: 저장, 새 테이블 추가, 엑셀 내보내기

3. 제목 바로 아래에는 대분류 탭을 둔다.
   - 대분류 탭은 사이트/업무 도메인 기준으로 구성한다.
   - 예: 전체 (149건), AI번역 (34건), 통독문서 (22건), 교육센터 (31건), 전시/행사 (18건), 시험사이트 (44건)
   - 대분류 탭도 반드시 `탭명 (0명)` 또는 `탭명 (0건)` 형식으로 카운트를 함께 표기한다.
   - 대분류 탭 자체에는 과한 설명을 넣지 않는다.
   - 대분류 탭 선택 시 하위 탭의 카운트와 테이블 내용이 바뀌게 보이도록 구성한다.

4. 대분류 탭 아래에는 하위 탭을 둔다.
   - 하위 탭의 라벨은 반드시 `탭명 (전체 00명)` 또는 데이터 성격에 맞게 `탭명 (전체 00건)` 형식으로 표기한다.
   - 예:
     - 전체 회원 (전체 45명)
     - 일반 회원 (전체 26명)
     - 기업 회원 (전체 8명)
     - 전문가 (전체 10명)
     - 결제 내역 (전체 28건)
     - 문의 내역 (전체 16건)
   - 카운트는 별도 배지로 떨어뜨리지 말고, 탭 텍스트 안에서 자연스럽게 읽히게 한다.
   - 단, 시각적으로는 괄호 안 카운트를 더 연한 색으로 처리해도 된다.

5. 하위 탭 아래에는 필터/검색 영역을 둔다.
   - 검색창 placeholder: 이름, 이메일, 회원번호, 연락처, 관리번호 검색
   - 필터는 기존 `design_system_major`의 검색필터 패턴을 그대로 사용한다.
   - 기본형은 `filter-bar`, `filter-group`, `filter-chip`, `filter-active` 구조를 따른다.
   - BO 고밀도 테이블 화면은 `dense-filter-card`, `dense-search-main`, `dense-filter-flow`, `dense-filter-group`, `filter-chip`, `dense-check`, `dense-active-filters` 구조를 우선 사용한다.
   - 필터는 2줄 이하로 조밀하게 배치하되, 체크박스 조건을 충분히 넣어 빈 영역 없이 꽉 차게 구성한다.
   - `DB페이지_v2(이거사용).html`의 검색조건을 빠뜨리지 않고 반영한다.
   - 회원 검색조건은 역할에 맞게 구조적으로 그룹핑하되, 화면 좌측에 `회원 기본`, `서비스/상태`, `운영/접근` 같은 그룹명을 별도 라벨로 노출하지 않는다.
   - 현재 BO 회원 검색조건 CSS 기준은 `.dense-filter-flow { display: flex; align-items: start; gap: 4px 7px; }`이다.
   - `.dense-filter-role-group`은 `display: contents`로 구조만 유지하고, 화면에는 그룹명 텍스트를 노출하지 않는다.
   - `.dense-filter-role-body`는 `height: 100%`와 `border-right: 1px solid var(--line-soft)`로 검색조건 컬럼 사이 세로 구분선을 만든다. 마지막 그룹의 border는 제거한다.
   - 각 조건 블록인 `.dense-filter-group`은 `display: flex`로 정렬한다. 내부에서 grid column을 한 번 더 만들지 않는다.
   - 조건명은 `.dense-filter-group legend { flex: 0 0 58px; }`로 고정하고, 옵션은 `.dense-filter-options { flex: 1 1 auto; display: flex; flex-wrap: wrap; }`로 배치한다.
   - `회원유형`, `서비스유형`, `성별`, `매칭상태`의 옵션 시작선이 어긋나지 않게 `legend` 58px 기준을 공통으로 유지한다.
   - 기간설정은 한 줄로 처리한다. `.filter-date-group .dense-filter-options`는 `flex-wrap: nowrap; overflow-x: auto;`를 사용한다.
   - 체크박스 조건: 회원유형, 성별, 구독상태, 서비스유형, 매칭상태, 전문가레벨, 시험종목, 결제상태, 가입채널, 알림, 기기
   - 셀렉트 조건: 카테고리 대분류, 중분류, 소분류
   - 기간 조건: 가입일순, 결제일순, 활동일순, 시험일순 + 시작일/종료일 + 오늘, 1주, 1개월, 3개월, 6개월, 1년 빠른 선택
   - 검색어 조건: 전체, 이름, 이메일, 관리번호, 연락처, 그룹명
   - 체크박스 후보: 일반회원, 정식회원, 기업회원, 전문가, M, F, 프리미엄, 스탠다드, 베이직, free, AI번역, 통독문서, 교육센터, 전시/행사, 시험사이트, 완료, 대기, 취소, 검토중, Lv.S, Lv.A, Lv.B, Lv.C, 번역시험, 영어교육, AI윤리, 문서관리, 정상, 미납, 환불, 정산대기, 이메일, 홈페이지, 구글, 추천, 관리자등록, 기관계약, SMS, 알림톡, PC, Mobile, Tablet
   - 필터 라벨은 13px 수준의 작은 텍스트로, 버튼/셀렉트는 높이 30px 내외로 유지한다.

6. 테이블 행 클릭 상세 Drawer
   - 테이블 행을 클릭하면 기존 `index.html`의 "회원목록 + 우측 Dense Drawer"처럼 목록 영역 오른쪽에 상세 패널이 붙어서 열린다.
   - 상세 버튼을 별도 컬럼으로 강제하지 말고, 행 전체를 클릭 가능한 대상으로 만든다.
   - Drawer는 오버레이 모달이 아니라 테이블과 같은 작업영역 안의 우측 컬럼이다.
   - 닫기 버튼, Escape 닫기, 선택 행 강조, 포커스 복귀를 포함한다.
   - Drawer 내부는 선택 행의 컬럼값을 요약 그리드와 입력 필드로 재구성한다.
   - 정보가 적은 행도 빈 공간을 크게 남기지 말고 운영 메모, 상태 변경, 저장 액션을 바로 붙인다.

테이블 설계 원칙:
- 테이블은 운영자가 한 화면에서 많은 행을 볼 수 있도록 조밀하게 만든다.
- BO 테이블의 기본 폰트 크기는 `8px`로 정의한다. `12px`를 기본값으로 쓰지 않는다.
- 테이블 헤더, 본문 셀, 보조 텍스트, badge, status, filter chip, checkbox label은 모두 `8px` 기준으로 맞춘다.
- 페이지 제목, 상단 업무명처럼 화면 위계를 만드는 텍스트만 예외적으로 더 크게 쓴다.
- 헤더 높이는 약 42px, 본문 행 높이는 약 48~52px 수준으로 유지한다.
- 날짜는 모든 곳에서 `YY-MM-DD` 형식으로 표기한다.
  - 예: 2026-02-09가 아니라 `26-02-09`
  - 날짜+시간이 꼭 필요한 활동 로그는 `YY-MM-DD HH:mm` 형식으로 표기한다.
- 짧은 정보이거나 같은 맥락의 정보는 반드시 한 컬럼 안에 묶는다.
  - 이름 + 나이 + 성별
  - 이메일 + 휴대폰
  - 가입일 + 가입채널
  - 최근 로그인 + 체류시간
  - 회원유형 + 구독상태
  - 전문가 레벨 + 평점
  - 매칭상태 + 진행상태
  - 결제상태 + 결제수단
- 컬럼 수를 늘리는 것보다 2줄 셀을 적극적으로 사용한다.
- 긴 메모, 주소, 제목, 카테고리명은 말줄임 처리하고 title/tooltip으로 전체 내용을 확인할 수 있게 보이게 한다.
- 상태값은 색상만으로 구분하지 말고 텍스트를 함께 표기한다.
- 체크박스 선택 컬럼과 관리번호/ID 컬럼은 좌측 고정처럼 보이게 한다.
```

## 핵심 규칙 요약

- 새 테이블은 기존 화면과 분리된 별도 페이지로 만든다.
- 제목 아래 대분류 탭을 먼저 두고, 대분류 탭은 `탭명 (00명/00건)`, 하위 탭은 `탭명 (전체 00명/00건)` 형식으로 수량을 표기한다.
- 검색필터는 새로 만들지 말고 기존 `filter-bar` 또는 `dense-filter-card` 패턴을 재사용한다.
- 테이블 행 클릭 시 기존 `index.html`의 "회원목록 + 우측 Dense Drawer"처럼 목록 영역 오른쪽 컬럼에 inline 상세 UI가 열린다.
- 테이블 목록에서는 묶음 컬럼을 유지하되, Drawer 상세에서는 `이름`, `나이`, `성별`, `이메일`, `휴대폰`, `주소`, `국가`처럼 편집 가능한 단위로 다시 분리한다.
- 회원 상세는 `docs/회원상세내용_이미지분석_프롬프트.md` 기준으로 좌측 회원기본정보 마스터와 우측 상담/시험/결제/정산 작업영역을 포함한다.
- Drawer 넓게보기 상태에서는 목록 테이블을 접고 상세 작업영역이 workspace 전체 폭을 사용한다.
- 선택 row는 배경색과 좌측 primary 인디케이터로 표시한다.
- 이름/나이/성별, 이메일/휴대폰, 가입일/채널처럼 짧거나 같은 맥락인 정보는 한 컬럼 안에 2줄로 묶는다.
- 모든 날짜는 `YY-MM-DD`, 활동 로그처럼 시간이 필요한 경우만 `YY-MM-DD HH:mm`으로 쓴다.
- 테이블은 BO 운영용으로 조밀하게 유지하고, 컬럼 수보다 정보 묶음과 말줄임 처리를 우선한다.
