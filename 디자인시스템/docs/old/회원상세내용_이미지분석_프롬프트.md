# 회원 상세내용 이미지 분석 기반 구현 프롬프트

기준 이미지: `table_design/회원_상세내용.png`

## 이미지 분석 요약

회원 상세 화면은 단순 조회 폼이 아니라 상담, 시험, 결제, 정산을 한 화면에서 처리하는 BO 운영 작업대다. 좌측에는 회원 기본정보 마스터가 고정되고, 우측에는 상담 진행 상태, 상담 메모, 상담 이력, 시험/납부 상세, 환급/정산 요약이 조밀하게 배치된다.

## 구현 프롬프트

```text
회원 목록의 행을 클릭했을 때 열리는 우측 inline Dense Drawer에 회원 상세 작업영역을 추가한다.

레이아웃:
- Drawer 기본 정보 섹션 아래에 `회원 상세 작업영역`을 둔다.
- 작업영역은 좌측 `회원기본정보 마스터`, 우측 `상담/시험/결제 작업 패널`의 2컬럼 구조로 만든다.
- 넓게보기 상태에서는 목록 테이블을 접고 Drawer가 작업영역 전체 폭을 사용한다.
- 오버레이 모달이 아니라 기존 목록과 같은 workspace 안에서 동작한다.

좌측 회원기본정보 마스터:
- 성명/영문, 아이디, 관리번호, 이메일, 휴대폰, 거주지/국가, 회원유형/구독상태, 가입일/채널, 전문가 레벨/평점을 한 줄 필드로 배치한다.
- 내국인/외국인 radio, 작업 목적 textarea, 출제/번역·수강등록·시험접수·전문가매칭 checkbox를 포함한다.
- 테이블의 묶음 컬럼은 상세에서 다시 이름, 나이, 성별, 이메일, 휴대폰, 주소, 국가처럼 편집 가능한 단위로 나눈다.

우측 상담/시험/결제 작업 패널:
- 상단 어두운 상태바에 `상담내용 [회원명 : 관리번호]`, 저장, 상담이력, 전환, FAIL 같은 운영 액션을 배치한다.
- 상담일시, 최근 상담일시, 총 상담 내역 보관 상태를 한 줄 상태 영역으로 표시한다.
- 상담종목, 교육, 수강정보를 select로 제공한다.
- 상담결과 선택은 미상담, 완료, LT예약, 다음등록, 실종, 부재, 기타 같은 세로 탭으로 배치한다.
- 상담 메모 textarea와 상담이력 로그를 나란히 둔다.
- LT 인터뷰 결과는 P/F/미 radio로 관리한다.
- 회원 납부 상세정보 및 증명서 실시간 결제·규칙 정산 현황은 작은 테이블로 표시한다.
- 하단에는 환급 진행 수수료 총액과 실시간 진행 내용 요약 textarea를 둔다.

시각 규칙:
- BO 테이블/상세 기본 글자 크기는 8px 기준으로 유지한다.
- 정보 밀도는 높게 유지하되, 섹션 경계는 얇은 border와 배경색 차이로만 구분한다.
- 제목/상태바만 더 진한 색을 사용하고, 입력 필드는 실무자가 바로 수정 가능한 형태로 둔다.
- 색상은 기존 BO 디자인 시스템 토큰을 우선 사용한다.
```
