# 상태/유형 칩 기준

기준 파일: `layout.css`, `table_design/table-design.css`

이 문서는 상태, 유형, 구독, 결제수단, 우선순위처럼 반복되는 값을 chip/pill로 표현하는 기준이다.

## 원칙

- 상태값은 색상 텍스트만 사용하지 않는다.
- chip은 텍스트, 상태 점 또는 아이콘, 연한 배경, border를 함께 가진다.
- 같은 의미는 같은 색상 체계를 사용한다.
- 텍스트는 항상 표시한다. 색상이나 아이콘만으로 상태를 전달하지 않는다.
- 테이블 셀 안에서는 chip 크기를 작게 유지한다.

## 상태 의미 체계

| 의미 | 클래스 | 예시 | 사용처 |
|---|---|---|---|
| 완료/성공/정상 | `status ok` | 완료, 결제완료, 배정완료, 성공 | 처리 완료, 결제 완료, 로그인 성공 |
| 대기/검토/정산 | `status wait` | 대기, 검수대기, 정산대기, 미납 | 확인 필요, 미처리, 보류 |
| 실패/취소/환불 | `status fail` | 취소, 환불, 환불완료 | 실패, 취소, 환불, 위험 |
| 진행/정보 | `status info` | 정상, 진행중, 운영중 | 진행 상태, 정보성 상태 |

## BO 테이블 상태 chip CSS 기준

```css
.status {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: 4px;
  min-height: 20px;
  padding: 2px 7px;
  border: 1px solid var(--status-line);
  border-radius: 999px;
  background: var(--status-bg);
  color: var(--status-color);
  font-size: var(--bo-font-size-base);
  font-weight: 700;
  white-space: nowrap;
}

.status::before {
  content: "";
  width: 5px;
  height: 5px;
  border-radius: 999px;
  background: currentColor;
}
```

## 유형/등급 badge 기준

| 의미 | 클래스 | 예시 |
|---|---|---|
| Primary 유형 | `badge blue` | 프리미엄, 주요 서비스 |
| 보조/전문가 | `badge violet` | 스탠다드, 전문가 |
| 위험/높음 | `badge red` | Lv.A, 높음 |
| 성공/완료 | `badge green` | 활성, 완료 |
| 기본/무료 | `badge gray` | free, 베이직 |
| 경고/보통 | `badge warn` | 보통, 확인필요 |

Badge도 원색 면을 피하고, 연한 배경 + 진한 텍스트 + border를 기본으로 한다.

## 결제상태/수단 표시

결제상태와 결제수단은 한 셀 안에 2줄로 묶는다.

```html
<td>
  <div class="cell">
    <span class="status ok">결제완료</span>
    <span class="sub">신용카드</span>
  </div>
</td>
```

`status + .sub`는 작은 보조 chip으로 표시한다.

## 매칭상태 표시

전문가명과 배정상태는 한 셀 안에서 이름 + 상태 chip으로 묶는다.

```html
<td>
  <div class="cell">
    <span class="main">이정희</span>
    <span class="status ok">배정완료</span>
  </div>
</td>
```

## 금지

- `완료`, `대기`, `환불`을 단순 초록/노랑/빨강 텍스트로만 표시하지 않는다.
- 같은 의미의 상태에 화면마다 다른 색을 쓰지 않는다.
- chip 내부에 긴 문장을 넣지 않는다.
- 상태 chip과 badge를 과도하게 크게 만들지 않는다.
- 보조값을 회색 텍스트로만 두어 상태값과 구분이 약해지게 하지 않는다.
