# 검색필터 기준

기준 파일: `index.html`, `layout.css`, `table_design/table-design.css`, `table_design/table-design.js`

이 문서는 사용자 화면 필터와 BO 고밀도 검색조건을 구분해서 정의한다.

## 사용자 화면 필터

사용자 화면은 과도하게 많은 조건을 한 번에 노출하지 않는다.

권장 구조:
- 키워드 검색
- 1~3개 주요 카테고리 chip
- 기간 또는 상태 필터가 필요하면 한 줄 추가
- 활성 필터는 `filter-tag`로 표시

```html
<div class="filter-bar">
  <button class="filter-chip" type="button" aria-pressed="true">전체</button>
  <button class="filter-chip" type="button" aria-pressed="false">접수중</button>
  <button class="filter-chip" type="button" aria-pressed="false">마감임박</button>
</div>
```

## BO 고밀도 검색조건

BO 테이블은 `dense-filter-card`를 기준으로 한다.

필수 구조:
- `dense-filter-card`
- `dense-search-main`
- `dense-filter-flow`
- `dense-filter-group`
- `dense-filter-options`
- `filter-chip`
- `dense-check`
- `dense-active-filters`

```html
<form class="dense-filter-card" data-dense-filter>
  <div class="dense-search-main">
    <label class="dense-search-field">
      <span>검색어</span>
      <input type="search" name="keyword" placeholder="이름, 이메일, 관리번호 검색">
    </label>
    <button class="btn primary" type="submit">검색</button>
    <button class="btn" type="reset">초기화</button>
  </div>

  <div class="dense-filter-flow">
    <fieldset class="dense-filter-group" data-single-choice>
      <legend>상태</legend>
      <div class="dense-filter-options">
        <button class="filter-chip" type="button" aria-pressed="true">전체</button>
        <button class="filter-chip" type="button" aria-pressed="false">완료</button>
        <button class="filter-chip" type="button" aria-pressed="false">대기</button>
      </div>
    </fieldset>
  </div>

  <div class="dense-active-filters" data-active-filters></div>
</form>
```

## 정렬 기준

- 조건명 `legend`는 고정 폭을 강제하지 않고, 필터 그룹 안에서 자연스럽게 흐르게 배치한다.
- 옵션 영역은 flex-wrap으로 배치한다.
- 조건 블록 사이에는 얇은 세로 구분선을 둘 수 있다.
- 기간 조건은 한 줄로 유지하고, 공간이 부족하면 가로 스크롤을 허용한다.
- 선택된 검색조건은 아래 활성 필터 영역에 tag로 요약한다.

## 권장 조건 세트

회원:
- 회원유형: 일반회원, 정식회원, 기업회원, 전문가, 관리자
- 성별: M, F
- 구독상태: 프리미엄, 스탠다드, 베이직, free
- 매칭상태: 완료, 대기, 취소, 검토중
- 결제상태: 정상, 미납, 환불, 정산대기
- 가입채널: 이메일, 홈페이지, 구글, 추천, 관리자등록, 기관계약

결제:
- 유형: 구독, 서비스의뢰, 시험응시, 문서구매
- 상태: 결제완료, 정산대기, 환불완료, 결제취소
- 수단: 카드, 계좌이체, 체크카드, 토스페이, 가상계좌

매칭:
- 유형, 카테고리, 언어, 분야, 배정상태, 진행상태, 결제상태

## 금지

- BO 필터를 큰 카드 여러 개로 나누지 않는다.
- 선택된 필터를 화면 오른쪽 끝으로 밀어내지 않는다.
- `전체` chip과 개별 chip의 선택 상태가 동시에 모호하게 보이게 하지 않는다.
- 조건명이 서로 다른 시작선에 걸려 옵션 정렬이 흐트러지게 하지 않는다.
