# 재사용 디자인 프롬프트 문서목록

이 폴더는 현재 `design_system_major`를 다른 프로젝트나 다른 화면 제작에 재사용하기 위한 기준 프롬프트 모음이다.

목표는 동일한 입력을 주었을 때 사용자 화면과 BO 테이블 화면이 같은 톤, 밀도, 컴포넌트 구조로 생성되게 하는 것이다.

## 폴더 구조

| 폴더 | 용도 | 먼저 볼 문서 |
|---|---|---|
| `사용자/` | 사용자 화면, 서비스 소개, 신청/안내 화면 제작 기준 | `사용자/00_문서목록.md` |
| `관리자/` | BO 관리자 화면, 대량 테이블, 검색필터, inline Drawer 제작 기준 | `관리자/00_문서목록.md` |

## 사용 순서

1. 사용자 화면을 만들 때는 `사용자/01_사용자화면_디자인시스템_기준.md`를 먼저 적용한다.
2. 관리자 화면 전체를 만들 때는 `관리자/03_BO_관리자화면_기준.md`를 먼저 적용한다.
3. 관리자 대시보드나 상세 작업 레이아웃은 `관리자/04_BO_관리자용_레이아웃_기준.md`를 함께 적용한다.
4. 관리자 대량 테이블 화면을 만들 때는 `관리자/02_BO_테이블레이아웃_기준.md`를 함께 적용한다.
5. 상태/유형/필터/Drawer가 필요한 관리자 화면은 `관리자/components` 문서를 함께 적용한다.
6. 테이블 행 선택 후 상세 작업을 처리하는 화면은 `관리자/components/테이블선택_Drawer_상세패널_기준.md`를 함께 적용한다.
7. 새 프로젝트에 복사할 때는 이 폴더 전체를 기준 프롬프트로 전달한다.

## 공통 금지

- 기준 파일에 없는 새로운 색상 팔레트를 임의로 만들지 않는다.
- 큰 카드, 과한 radius, 넓은 여백, 마케팅식 장식을 BO 테이블 화면에 적용하지 않는다.
- 상태값을 색상 텍스트만으로 표시하지 않는다.
- 날짜와 카운트 표기 형식을 화면마다 다르게 쓰지 않는다.
- 사용자 화면과 관리자 화면의 밀도, 글자 크기, 정보 구조를 섞지 않는다.
