# 세 사이트 레이아웃/통일성 분석

분석 대상:

- https://hu-tec.github.io/TESOL-v3
- https://aite-test.imweb.me/index
- https://hu-tec.github.io/ai-teacher

분석 목적: 세 사이트를 같은 계열의 교육/자격 랜딩페이지처럼 보이게 만들기 위한 정보 구조, 레이아웃, 스타일 공통 기준 도출.

## 1. 전체 요약

세 사이트는 모두 교육/자격/AI 역량을 다루며, 신뢰성, 과정 소개, 신청 유도, 상담 연결이 핵심 목표다. 다만 현재 표현 방식은 서로 다르다.

| 사이트 | 성격 | 현재 강점 | 통일 시 조정 포인트 |
|---|---|---|---|
| TESOL-v3 | 교육기관/과정형 멀티페이지 | GNB, 과정 상세, 입학/커뮤니티/신청 플로우가 체계적 | 전체 사이트의 기준 레이아웃으로 삼기 좋음 |
| AITe Imweb | 자격시험 원페이지/마케팅형 | 시험 종류, 신뢰 근거, 후기, 프로세스가 풍부함 | 섹션이 길고 중복 문구가 있어 구조 정리가 필요 |
| ai-teacher | AI 강사 양성과정형 멀티페이지 | TESOL-v3와 CSS 토큰/레이아웃이 거의 동일 | TESOL-v3와 통합 브랜드 규칙 적용이 쉬움 |

권장 방향: `TESOL-v3 / ai-teacher`의 1200px 컨테이너, sticky GNB, 와인 레드 포인트, 카드/그리드 구조를 공통 디자인 시스템으로 잡고, `AITe Imweb`의 풍부한 마케팅 섹션을 같은 컴포넌트 규칙으로 재배치한다.

## 2. 사이트별 구조 분석

### 2.1 TESOL-v3

기술/구조:

- React/Vite SPA.
- 기본 경로는 `/TESOL-v3/`.
- 라우트: 홈, 교육원 소개, 입학 안내, 과정 상세 5종, 커뮤니티, 문의, 로그인, 회원가입, 신청서.
- 공통 UI: sticky header, dropdown GNB, footer, card, table, process flow, form panel.

상단 내비게이션:

- 로고: `A. TESOL`
- 서브타이틀: `국제 영어교사 양성과정`
- 메뉴: 홈, 교육원 소개, 입학안내, 과목 소개, 커뮤니티, 문의
- 과목 소개 드롭다운: TESOL 영어교육, AI 번역·통역, AI 프롬프트, AI 윤리, ITT 정통 통번역
- 데스크톱 CTA: 수강 신청
- 모바일: 햄버거 메뉴 + 하단 신청 버튼

홈 섹션 흐름:

1. Hero
   - 어두운 배경, 와인 레드 하이라이트, 중앙 정렬
   - 키 메시지: 글로벌 TESOL, 영어교육/미래교육 강조
   - CTA: 교육과정 보기, 수강 신청하기, 1:1 문의하기
2. 브랜드 소개
   - `글로벌 교육의 기준, A. TESOL`
   - 교육기관의 미션/신뢰감 전달
3. 과정 카드 그리드
   - `미래를 준비하는 5가지 전문 과정`
   - 5개 과정으로 분류
4. 맞춤형 교육 서비스
   - 레벨테스트, 설명회, 상담, 신청 등 빠른 액션
5. 교육 네트워크
   - 대학/협약처/해외 제휴 기관 수치형 신뢰 근거
6. 하단 CTA
   - 무료 수강상담, 입학설명회 신청

레이아웃 특징:

- 기본 컨테이너 `max-width: 1200px`, 좌우 `20px`.
- 주요 섹션은 `py-16`, `py-24` 수준의 넓은 세로 간격.
- 카드: 흰 배경, 12px radius, 얕은 shadow, 1px border.
- 과정 상세: 상단 page header → LNB/tab → 과정 헤더 → 이미지+정보 2열 → 설명/표/커리큘럼.
- 신청서: `max-width: 56rem`, 2단계 progress, 2열 form grid.

시각 스타일:

- 폰트: Pretendard.
- 메인 컬러: `#9b2226`, 일부 신청 폼은 `#8B1A2B`.
- 배경: `#f9fafb`, 보조 배경 `#f3f4f6`, 본문 `#1f2937`, 보조 텍스트 `#6b7280`.
- Hero는 검정/딥 와인 그라데이션과 grid texture 사용.

### 2.2 AITe Imweb

기술/구조:

- Imweb 기반 사이트.
- 공개 페이지 텍스트 구조상 원페이지 성격이 강함.
- GNB는 시험 소개, 자격 관리, 게시판, 고객지원, 협회 소개 중심.

상단 내비게이션:

- 메뉴: 시험 소개, 자격 관리, 게시판, 고객지원, IITA 협회 소개
- 자격 관리 하위: AITe 번역전문가, AITe 프롬프트 전문가, ITT 통번역, AI 윤리 시험
- 로그인/회원가입/마이페이지 요소 포함

홈 섹션 흐름:

1. Hero
   - 문구: `AI시대 당신의 역량을 공식 인증합니다`
   - 보조: `25년 전통과 AI 미래 자격 지금 바로 도전하세요`
   - CTA: 문의하기
2. Exam Types
   - AI 역량 인증 시험 소개
   - 4개 시험: AI 번역, AI 프롬프트, AI 윤리, ITT 전통 통번역
   - 각 카드에 대상 해시태그와 시험 안내 링크
3. 문제 제기/가치 제안
   - `AI없이는 하루도 살 수 없는 시대`
   - AI 활용 능력 검증 필요성 강조
4. 자격증 카테고리 카드
   - 번역, 웹툰 번역, 유튜브 번역, 동시통역, 개발, 프롬프트 언어 등
   - 현재 반복 노출되는 카드가 있어 정리 필요
5. 신뢰/차별화 섹션
   - 공인 자격증, 법무부/과기부 인증, 25년 전통, 30만 명 응시 등
   - 실무형 역량, 종합 자격시험, 전문가 검증 등
6. 후기 섹션
   - 취업, 승진/프로젝트 리더, 프리랜서 신뢰도, AI 번역 활용 사례
7. 프로세스 섹션
   - 회원가입 → 시험 선택 → 급수 선택 → 결제 → 응시확인서 출력
8. Footer
   - 전화, 운영시간, 회사 정보, 이메일, copyright

레이아웃 특징:

- 마케팅 원페이지에 가까운 긴 스크롤 구조.
- 반복 카드, 이미지 카드, 후기 카드, 프로세스 step이 많이 배치됨.
- TESOL-v3/ai-teacher보다 섹션 위계가 덜 정리되어 있어, `Hero → 시험유형 → 신뢰근거 → 후기 → 프로세스 → CTA`로 압축하면 통일성이 올라감.

시각/콘텐츠 특징:

- AI 자격/인증의 즉시성, 공신력, 실무성을 강조.
- 해시태그형 대상 표현을 적극 사용.
- 텍스트 중 오탈자/표현 정리 필요: 예) `Skiles`, `아우루는`, 띄어쓰기/문장 반복.

### 2.3 ai-teacher

기술/구조:

- React/Vite SPA.
- 실제 배포 HTML은 `/AI_teach-v2/` 자산을 사용.
- 라우트: 홈, 교육원 소개, 과정 소개, 수강 안내 하위 페이지, 커뮤니티, 문의.
- CSS 토큰과 컴포넌트 구조가 TESOL-v3와 거의 같다.

상단 내비게이션 추정 구조:

- 홈
- 교육원 소개
- 과정 소개
- 수강 안내: 등록 안내, 일정 안내, 수강혜택, 환불 안내, 증명서 출력
- 커뮤니티
- 1:1 문의하기

주요 페이지/섹션:

1. 홈
   - AI 강사 양성과정 중심 Hero
   - 과정 소개/수강 신청 CTA
2. 교육원 소개
   - 교육기관 신뢰, AI 교육 필요성, 운영 철학
3. 과정 소개
   - AI 방과후 교사, 학원·공부방 강사, 온라인 전문 강사, 콘텐츠 기획 제작 등 분야별 과정
4. 수강 안내
   - 등록 절차
   - 교육 일정표
   - 수강 혜택 카드 6개
   - 환불 기준 표
   - 증명서 출력 절차
5. 커뮤니티
   - 공지사항, FAQ, 수강후기, 자료실 탭
6. 문의
   - 상담 분야, 이름, 연락처, 문의 내용, 개인정보 동의

레이아웃 특징:

- TESOL-v3와 동일한 `container`, `card`, `page-header`, `sub-tabs`, `schedule-table`, `grid col-2/3/4` 구조.
- `page-header`는 딥 그레이/블랙 그라데이션에 중앙 제목.
- 수강혜택은 3열 카드, 환불/증명서/일정은 표와 step flow 중심.
- 문의 폼은 800px 내외의 단일 카드에 집중.

시각 스타일:

- Pretendard.
- 메인 컬러 `#9b2226`.
- 밝은 회색 배경, 흰 카드, 와인 레드 CTA.
- 전체적으로 TESOL-v3와 가장 통일하기 쉬움.

## 3. 공통 레이아웃 패턴

세 사이트를 하나의 계열로 보이게 하려면 아래 구조를 공통 템플릿으로 잡는 것이 좋다.

### 3.1 공통 페이지 골격

```text
Sticky Header
  Logo + 서비스명
  주요 메뉴
  우측 CTA

Main
  Hero 또는 Page Header
  핵심 카테고리/과정/시험 카드
  상세 설명 또는 비교 영역
  신뢰 근거
  프로세스/일정/커리큘럼
  후기/FAQ/공지
  상담/신청 CTA

Footer
  회사 정보
  교육/시험 메뉴
  지원 서비스
  연락처/운영시간
```

### 3.2 홈/랜딩 공통 섹션 순서

1. Hero: 브랜드/과정/시험명을 첫 화면에서 크게 노출
2. Category Cards: 과정 또는 시험 유형 3~6개
3. Why/Value: 왜 필요한지, 무엇을 얻는지
4. Trust Proof: 인증, 연혁, 응시자 수, 협약, 교수진
5. Process: 신청/응시/수강 절차
6. Schedule 또는 Curriculum: 실제 행동에 필요한 정보
7. Reviews/Stories: 결과 중심 후기
8. Final CTA: 상담/신청/문의

### 3.3 서브페이지 공통 구조

```text
Page Header
  h1: 페이지명
  breadcrumb 또는 짧은 설명

Local Nav / Tabs
  같은 카테고리 내 하위 메뉴

Content Sections
  카드, 표, 리스트, step flow

CTA Strip
  상담 또는 신청 연결
```

## 4. 통일 디자인 가이드

### 4.1 컬러

현재 TESOL-v3와 ai-teacher가 이미 공유하는 토큰을 기준으로 삼는다.

| 역할 | 권장값 | 용도 |
|---|---|---|
| Primary | `#9b2226` | CTA, active, 핵심 강조 |
| Primary Hover | `#b82a30` | 버튼 hover |
| Deep Text | `#1f2937` | 제목, footer 배경 |
| Body Text | `#374151` 또는 `#4b5563` | 본문 |
| Muted Text | `#6b7280` | 보조 설명 |
| Page BG | `#f9fafb` | 전체 배경 |
| Section BG | `#f3f4f6` | 보조 영역 |
| Border | `#e5e7eb` | 카드/표 경계 |
| White | `#ffffff` | 카드/본문 영역 |

AITe 사이트도 이 색상으로 맞추면 세 사이트가 같은 브랜드 계열로 보인다.

### 4.2 타이포그래피

- 기본 폰트: Pretendard.
- Hero H1: 48~64px desktop, 34~42px mobile.
- Page Header H1: 36~40px desktop, 28~32px mobile.
- Section Title: 26~32px.
- Card Title: 18~22px.
- Body: 16px, line-height 1.6.
- 보조 영문/eyebrow: 12~14px, letter spacing은 과도하게 쓰지 않기.

### 4.3 컨테이너/간격

- 공통 container: `max-width: 1200px; padding: 0 20px; margin: 0 auto;`
- Section padding: desktop `80~96px`, mobile `48~64px`.
- 카드 grid gap: `24~32px`.
- 카드 radius: 8~12px. 너무 둥근 24~32px 카드는 특별한 강조 영역에만 사용.
- 버튼 radius: 6~8px.

### 4.4 컴포넌트 규칙

공통으로 재사용할 컴포넌트:

- `Header`
- `Footer`
- `Hero`
- `PageHeader`
- `SectionTitle`
- `CourseCard` / `ExamCard`
- `TrustCard`
- `ProcessStep`
- `ScheduleTable`
- `ReviewCard`
- `CTASection`
- `ContactForm`

AITe의 긴 원페이지도 위 컴포넌트로 재조립하면 TESOL-v3/ai-teacher와 구조가 맞는다.

## 5. AITe 사이트 재구성 권장안

현재 AITe는 정보량이 많지만, 반복 섹션이 많아 메시지가 분산된다. 아래 순서로 재구성하는 것을 권장한다.

```text
Header
Hero
  "AI시대 당신의 역량을 공식 인증합니다"
  CTA: 시험 안내 보기 / 문의하기

Exam Types
  AI 번역
  AI 프롬프트
  AI 윤리
  ITT 통번역

Why AITe
  AI 활용 능력 검증
  실무형 평가
  공신력 있는 인증

Trust Proof
  25년 전통
  30만 명 응시
  법무부/과기부 인증
  국내 유일 AI+통번역 종합 자격

Career Cards
  번역
  웹툰/유튜브
  통역
  프롬프트

Reviews
  취업/커리어/프리랜서/업무 효율 사례 4개

Process
  회원가입 → 시험 선택 → 급수 선택 → 결제 → 응시확인서 출력

Final CTA
  전화/문의/시험 접수

Footer
```

정리 기준:

- `Exam Types`는 4개로 고정한다.
- 반복되는 자격증 카드 6개 세트는 1회만 사용한다.
- 인증/공신력 문구는 `Trust Proof`로 묶는다.
- 후기 카드는 3~4개만 노출하고 나머지는 상세/게시판으로 이동한다.
- 프로세스는 TESOL-v3/ai-teacher의 `flow-grid`와 동일한 형태로 통일한다.

## 6. 세 사이트 통일 메뉴 제안

각 사이트의 주제는 다르지만 메뉴 구조는 비슷하게 맞출 수 있다.

| 역할 | TESOL-v3 | AITe | ai-teacher |
|---|---|---|---|
| 홈 | 홈 | 홈 | 홈 |
| 소개 | 교육원 소개 | 시험 소개/협회 소개 | 교육원 소개 |
| 주요 상품 | 과목 소개 | 자격 관리 | 과정 소개 |
| 이용 안내 | 입학 안내 | 응시 안내/프로세스 | 수강 안내 |
| 커뮤니티 | 커뮤니티 | 게시판 | 커뮤니티 |
| 문의 | 문의 | 고객지원 | 1:1 문의 |
| CTA | 수강 신청 | 시험 접수/문의 | 수강 신청 |

권장 GNB 명칭:

## 8. 최종 권장 기준
