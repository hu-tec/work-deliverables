# 원본 사이트 디자인 통일 작업 지침

## 목적

원본 사이트를 새 디자인 시스템에 맞춰 바꿀 때, 단일 랜딩페이지로 재구성하지 않고 원본의 정보구조, 메뉴, 서브페이지, 문구를 유지한 상태에서 디자인만 통일한다.

이번 작업의 핵심 목표는 "저 사이트가 우리가 만든 틀을 입으면 어떻게 보이는지"를 확인하는 것이다. 따라서 콘텐츠 기획, 문구 재작성, 섹션 재배치, 메뉴 축소는 작업 범위가 아니다.

## 반드시 지켜야 할 원칙

1. 콘텐츠는 원본 그대로 사용한다.
   - 제목, 본문, 메뉴명, CTA 문구, 표 항목, 푸터 정보는 임의로 바꾸지 않는다.
   - 맞춤법이나 어색한 문장도 사용자가 요청하기 전에는 수정하지 않는다.
   - 요약본으로 대체하지 않는다.

2. 정보구조를 유지한다.
   - 원본 상단 메뉴를 그대로 옮긴다.
   - 드롭다운 메뉴와 하위 페이지도 빠짐없이 만든다.
   - 폴더 메뉴는 첫 번째 하위 페이지 또는 별도 인덱스 페이지로 연결한다.

3. 단일 랜딩페이지로 합치지 않는다.
   - `시험 소개`, `AITe 번역전문가`, `AITe 프롬프트 전문가`, `ITT 통번역`, `AI 윤리 시험`, `게시판`, `고객지원`, `협회 소개`, `협회 구성`, `협회 연혁`은 각각 별도 페이지로 만든다.
   - 메인 페이지에 모든 내용을 몰아넣지 않는다.

4. 디자인만 통일한다.
   - 기존 `site-template`의 레이아웃, 헤더, 카드, 표, CTA, 푸터 스타일을 적용한다.
   - 색상은 원본 로고 색상과 기존 템플릿 디자인 토큰을 조합한다.
   - 콘텐츠 순서와 의미는 원본 기준을 우선한다.

## 작업 순서

1. 원본 사이트 수집
   - 각 원본 URL의 HTML을 저장한다.
   - 메뉴명, 페이지 제목, 본문 텍스트, 표, 이미지 URL, 푸터 정보를 추출한다.
   - 추출 결과를 페이지별로 분리해서 기록한다.

2. 콘텐츠 잠금
   - 추출한 텍스트를 먼저 `content-source.md` 또는 데이터 파일에 원문 그대로 보관한다.
   - 디자인 작업 중에는 이 원문을 수정하지 않는다.
   - 변경이 필요하면 별도 "수정 제안"으로만 남긴다.

3. 페이지 생성
   - 메뉴 매핑표 기준으로 모든 로컬 HTML 파일을 만든다.
   - 각 페이지는 같은 헤더/푸터를 공유하되, 본문은 해당 원본 페이지 콘텐츠만 사용한다.
   - 드롭다운과 현재 페이지 active 상태를 구현한다.

4. 디자인 적용
   - 기존 템플릿의 공통 컴포넌트를 사용한다.
   - 텍스트 섹션, 카드 섹션, 표 섹션, CTA 섹션으로만 시각 구조를 정리한다.
   - 원본 콘텐츠를 새로 쓰거나 마케팅 카피로 대체하지 않는다.

5. 검증
   - 모든 메뉴 링크가 로컬 파일로 연결되는지 확인한다.
   - 원본 메뉴에 있던 하위 페이지가 누락되지 않았는지 확인한다.
   - 각 페이지의 핵심 제목과 CTA가 원본과 일치하는지 확인한다.
   - 외부 이미지 의존성이 있으면 로컬 `assets/`로 저장하거나 명확히 기록한다.

## 금지 사항

- 원본의 여러 페이지를 하나의 랜딩페이지로 합치기
- 원본 메뉴를 임의로 줄이기
- "보기 좋게" 문구를 새로 쓰기
- 원본에 없는 섹션을 추가해 핵심처럼 보이게 하기
- 빈 페이지를 임의 콘텐츠로 채우기
- 게시판/고객지원처럼 콘텐츠가 적은 페이지를 다른 내용으로 대체하기

## 완료 기준

- 로컬 폴더에 원본 메뉴 수와 동일한 페이지가 존재한다.
- 상단 메뉴와 드롭다운이 원본 구조와 일치한다.
- 각 페이지의 제목, 주요 본문, 표/카드 항목, CTA, 푸터 정보가 원본 기반이다.
- 디자인은 기존 템플릿 톤으로 통일되어 있다.
- 사용자가 "콘텐츠는 그대로이고 디자인만 바뀌었다"고 판단할 수 있다.

## 다음 수정 작업 메모

현재 `aite-translation-landing` 폴더는 단일 랜딩 방식으로 시작된 흔적이 있으므로, 다음 수정에서는 먼저 콘텐츠 데이터와 페이지 파일을 원본 기준으로 재검토해야 한다.

우선순위는 다음과 같다.

1. 원본 텍스트 전체 추출본을 별도 파일로 보관
2. 누락된 서브페이지 콘텐츠 보강
3. 각 페이지별 본문을 원본 순서대로 재배치
4. `site-template`의 디자인 컴포넌트와 색상 토큰만 적용
5. 최종 링크 검증
