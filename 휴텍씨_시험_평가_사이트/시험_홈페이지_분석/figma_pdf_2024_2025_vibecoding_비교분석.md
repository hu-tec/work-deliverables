# figma_pdf 2024 / 2025 / vibecoding 비교 분석

작성일: 2026-05-19  
분석 기준: `figma_pdf/2024`, `figma_pdf/2025`, `figma_pdf/vibecoding` 폴더의 파일명, 파일 수, 대표 이미지 확인 기준  
주의: PDF 내부 IA/텍스트 전문은 아직 별도 IA 분석 결과가 반영되지 않았다. 이 문서는 구현 순서와 공통/차이 범위를 잡기 위한 1차 비교 문서다.

## 결론 요약

`2024`는 전체 서비스의 기준 버전으로 봐야 한다. 회원, 시험, 문제, 출제, 채점, 결제, 정산, 운영, 게시판, 고객지원, 마이페이지, 응시자 화면까지 가장 넓은 범위를 포함한다.

`2025`는 전체 재작성본이라기보다 2024 구조 위에 UX/UI 참고 화면과 일부 신규/개편 화면을 얹은 버전에 가깝다. 특히 `양식관리`, `템플릿 양식 관리자`, `창작작업실`, `메타번역`, `파일업로드`, 관리자 상세 화면이 새로 보인다. 이 중 `창작작업실`은 현재 기준에서 확정 구현 기능이라기보다 참고 UX/UI 자료로 보는 것이 안전하다.

`vibecoding`은 독립된 전체 서비스 버전이 아니라 `파일업로드`와 `구분` 화면 묶음 중심의 별도 시안/프로토타입으로 보는 것이 타당하다. 2025와 `기본화면_파일업로드.jpg`, `Frame 758532606.jpg`, `구분-4.pdf`, `구분-5.pdf`, 체크박스 리소스를 공유하는 화면 계열이다.

따라서 구현은 `2024`를 베이스 IA와 공통 컴포넌트 기준으로 먼저 만들고, 이후 `2025`에서 추가된 신규 업무 화면을 확장하며, 마지막으로 `vibecoding`의 파일업로드/구분 흐름을 별도 기능 단위로 붙이는 순서가 적절하다.

## 파일 범위 비교

| 구분 | 전체 파일 수 | PDF | PNG | JPG | SVG | 의미 있는 화면 수 추정 |
|---|---:|---:|---:|---:|---:|---:|
| 2024 | 215 | 137 | 73 | 0 | 5 | 약 153 |
| 2025 | 205 | 19 | 183 | 3 | 0 | 약 27 |
| vibecoding | 64 | 16 | 46 | 2 | 0 | 약 17 |

의미 있는 화면 수는 `CheckBox`, `Vector`, `Frame`, `Group`, `Rectangle`, `image` 같은 리소스성 파일을 제외하고 계산한 값이다. 특히 2025는 `CheckBox` PNG가 174개라 전체 파일 수만 보면 2024와 비슷하지만, 실제 화면 범위는 훨씬 좁다.

## 공통점

세 버전 모두 관리자형 화면의 기본 골격을 공유한다. 좌측 아이콘 사이드바, 상단 검색/사용자 영역, 본문 테이블, 체크박스 필터, 저장/검색/파일 관련 버튼 구조가 반복된다.

공통 화면명 또는 계열은 다음과 같다.

| 공통 항목 | 포함 버전 | 해석 |
|---|---|---|
| `CheckBox*.png` | 2024, 2025, vibecoding | 체크박스 상태별 리소스. 공통 컴포넌트화 대상 |
| `구분-4.pdf`, `구분-5.pdf` | 2024, 2025, vibecoding | 구분/분류 관련 화면 계열 |
| `시험관리_신규.pdf` | 2024, 2025 | 시험 생성/등록 플로우가 유지됨 |
| `채점관리_단답형상세.pdf` | 2024, 2025 | 단답형 채점 상세 플로우 유지 |
| `출제관리_프롬프트상세.pdf` | 2024, 2025 | 프롬프트 출제 상세 플로우 유지 |

## 2024 특징

2024는 서비스의 원형 IA와 전체 업무 플로우가 가장 많이 들어 있다.

주요 범위:

- 관리자 홈 및 전체 메뉴 구조
- 회원관리: 수험생, 출제자, 채점자, 관리자
- 시험관리: 일정, 응시, 결과, 자격증
- 문제/출제관리: 번역, 음성/영상, 웹툰번역, 프롬프트, 단답형
- 채점관리: 번역 채점, 프롬프트 채점, 단답형 채점, 피드백
- 응시생 화면: 시험 시작, 시험 중, 제출, 종료, 프롬프트/단답형 응시
- 결제/정산: 결제, 포인트 결제, 결제내역, 정산관리, 결제 미승인/완료
- 운영관리: 알림창, 게시판/FAQ, 고객지원, 블로그, 사이트관리
- 계정/인증: 로그인, 로그아웃, 회원가입, 아이디/비밀번호 찾기, 휴대폰/본인인증
- 전문가/프로필/활동관리

구현 관점에서는 2024가 DB 모델, 권한, 라우팅, 공통 레이아웃, 목록/상세/등록 패턴을 정의하는 기준이다. 2024를 먼저 만들면 이후 2025와 vibecoding은 추가 화면 또는 변형 화면으로 붙일 수 있다.

## 2025 특징

2025는 화면 범위는 줄었지만 2024에 없거나 이름이 달라진 신규 기능이 많다.

2025에서 새로 보이는 항목:

- `양식관리.pdf`
- `양식관리_신규.pdf`
- `양식관리_프롬프트첫화면.pdf`
- `양식관리_프롬프트첫화면-1.pdf`
- `템플릿 양식 관리자.pdf`
- `창작작업실01.png`
- `메타번역.png`
- `기본화면_파일업로드.jpg`
- `사용자_영역설명.png`
- `목록_전문가프로필_상세입력.png`
- `관리자_시험관리_신규.pdf`
- `관리자_자격증관리_상세.pdf`
- `관리자_카테고리 관리.pdf`
- `어드민/대시보드_01.pdf`
- `마이페이지 1920해상도 참고.pdf`

대표 이미지 기준으로 `창작작업실01.png`는 2024의 시험/채점 중심 관리자 IA와 성격이 다르다. 상단에 `창작작업실`, `요금제`, `고객센터`, `회사소개`가 있고, 본문은 Youtube/문서/리서치&분석/프로그래밍/이미지·디자인/음악/음성/기타 탭, AI 선택, 프롬프트/파일 입력, 임시저장/카드/적용 액션을 포함한다. 다만 이 화면은 시험 플랫폼의 필수 구현 화면이라기보다, 작업형 화면을 설계할 때 참고할 UX/UI 레퍼런스로 분류한다.

즉 2025는 기존 시험 플랫폼에 `양식`, `템플릿`, `파일 업로드`, `메타 번역`, 작업형 UX/UI 참고 화면이 추가된 버전으로 보는 것이 맞다. `창작작업실`은 참고 자료로 두고 구현 범위는 별도 확정이 필요하다.

## vibecoding 특징

vibecoding은 다음 파일로 구성된다.

- `기본화면_파일업로드.jpg`
- `Frame 758532606.jpg`
- `구분.pdf`
- `구분-1.pdf` ~ `구분-14.pdf`
- `제목 없음.pdf`
- `CheckBox*.png`

화면명이 거의 `구분`과 `파일업로드`에 집중되어 있으므로 전체 서비스가 아니라 특정 업무 흐름의 상세 시안으로 판단된다. 2025와 파일명이 겹치는 항목이 많아 2025 신규 기능의 파생/실험 버전일 가능성이 높다.

구현 관점에서는 2024 베이스나 2025 신규 IA와 분리해서, `파일 업로드 기반 대량 처리/분류/구분 관리` 기능으로 모듈화하는 편이 좋다.

## 주요 차이점

| 비교 항목 | 2024 | 2025 | vibecoding |
|---|---|---|---|
| 역할 | 전체 서비스 원형 | 2024 기반 신규/개편 화면 | 파일업로드/구분 특화 시안 |
| IA 범위 | 가장 넓음 | 일부 핵심 화면만 존재 | 매우 좁음 |
| 주요 도메인 | 시험, 응시, 출제, 채점, 결제, 정산, 운영 | 양식, 템플릿, 메타번역, 파일업로드, 작업형 UX/UI 참고 | 파일업로드, 구분/분류 |
| 사용자 역할 | 관리자, 수험생, 출제자, 채점자 | 관리자 중심 + 창작 작업 사용자 | 관리자형 대량 처리 화면 중심 |
| 화면 스타일 | 시험 플랫폼/관리자 시스템 | 관리자 화면 + 작업형 UX/UI 참고 화면 혼합 | 2025 관리자 파일업로드 화면 계열 |
| 구현 우선순위 | 1순위 베이스 | 2순위 확장 | 3순위 기능 모듈 |

## 구현 순서 제안

1. 2024 공통 기반 구현

- 라우팅 구조, 공통 관리자 레이아웃, 좌측/상단 네비게이션
- 목록/상세/등록/수정/팝업 패턴
- 회원, 시험, 출제, 채점, 결제, 정산, 고객지원 등 주요 도메인
- 응시생 화면은 관리자 화면과 레이아웃이 다르므로 별도 영역으로 분리

2. 2025 신규 화면 확장

- 2024에 이미 있는 `시험관리_신규`, `채점관리_단답형상세`, `출제관리_프롬프트상세`, `회원관리_상세_전체`는 기존 구현을 재사용하고 차이만 반영
- `양식관리`, `템플릿 양식 관리자`, `메타번역`, `파일업로드`는 신규/개편 후보로 검토하고, `창작작업실`은 UX/UI 참고 자료로 분류
- 2025의 체크박스/필터가 더 많으므로 공통 체크박스/필터 컴포넌트는 2025 요구까지 견딜 수 있게 설계

3. vibecoding 기능 모듈 추가

- `기본화면_파일업로드`를 2025 파일업로드 화면과 비교해 공통 레이아웃 재사용
- `구분-1` ~ `구분-14`는 상태/단계별 화면일 가능성이 높으므로 IA 분석 완료 후 플로우로 묶기
- 전체 메뉴에 넣기보다 `파일 업로드 > 구분/분류 처리` 같은 하위 기능으로 붙이는 것이 안전

## 공통 컴포넌트 후보

- 관리자 레이아웃: 사이드바, 상단 검색, 사용자/로그아웃 영역
- 테이블: 고밀도 데이터 테이블, 가로 스크롤, 상태 배지, 선택 체크박스
- 필터 영역: 다중 체크박스, 셀렉트, 날짜 범위, 검색어 입력
- 액션 버튼: 저장, 검색, 파일올리기, 파일다운로드, 엑셀파일생성
- 탭: 회원관리/활동관리/결제관리/1:1문의/통계, 창작작업실 카테고리 탭
- 상세 화면: 기본 정보, 상태, 첨부/파일, 피드백, 저장
- 팝업/모달: 저장 확인, 문의, 평가서, 시험접수 변경/취소
- 파일 업로드: 업로드 버튼, 첨부 파일 목록, 대량 처리 결과 테이블

## IA 분석 결과가 오면 확인할 항목

- 2024 `메뉴 구조.png`의 메뉴명이 실제 라우팅 기준인지 확인
- 2025 신규 화면이 2024 메뉴 어디에 편입되는지 확인
- `양식관리`와 `템플릿 양식 관리자`의 관계 확인
- `창작작업실`은 현재 참고 자료로 보고, 실제 사용자 서비스 기능으로 구현할지 별도 확인
- vibecoding `구분` PDF 15개가 같은 화면의 상태 변화인지, 단계별 플로우인지 확인
- 2024와 2025의 동일 이름 파일이 실제로 UI만 변경된 것인지, 기능 요구사항까지 바뀐 것인지 확인

## 1차 판단

현재 자료만 기준으로 하면 `2024 = 베이스`, `2025 = 신규 기능/개편`, `vibecoding = 파일업로드/구분 특화 모듈`이다. 다른 세션의 IA 분석이 끝난 뒤에는 이 문서에 메뉴 트리와 화면별 구현 체크리스트를 붙이면 바로 2024부터 순차 구현 계획으로 전환할 수 있다.

## UX/UI 관점 보완

세 자료는 같은 수준의 산출물이 아니다. 2024는 전체 서비스의 운영형 관리자/시험 화면, 2025는 양식 기반 화면과 작업형 UX/UI 참고 화면, vibecoding은 대량 테이블 관리 화면이다. 따라서 UI를 하나의 방식으로 통일하기보다 업무 성격별로 다른 밀도와 조작 방식을 적용해야 한다.

| 구분 | 화면 성격 | UX/UI 방향 |
|---|---|---|
| 2024 | 시험 플랫폼 전체 운영 | 안정적이고 예측 가능한 업무형 UI. 목록/상세/등록/팝업 패턴을 일관되게 사용한다. |
| 2025 | 번역, 양식/템플릿, 작업형 UX/UI 참고 | 사용자가 작업 조건을 선택하고 결과를 확인하는 화면 패턴 참고. 좌측 설정, 중앙 입력, 우측 결과/옵션 구조를 참고한다. |
| vibecoding | 관리자 대량 조회/처리 | 고밀도 데이터 그리드 UI. 컬럼 그룹, 고정 열, 저장된 필터, 엑셀 처리 이력이 중요하다. |

| 원칙 | 설명 |

| 역할별 첫 화면 분리 | 수험생, 출제자, 채점자, 관리자는 서로 다른 첫 화면을 가져야 한다. 작업형 참고 화면은 별도 기능 확정 후 적용한다. |
| 목록과 작업 화면 분리 | 조회용 목록 화면과 실제 작성/채점/번역 작업 화면은 레이아웃 밀도가 달라야 한다. |
| 상태 기반 행동 제공 | 사용자가 다음에 할 수 있는 행동만 버튼으로 노출한다. 예: 접수 전에는 `접수하기`, 결제 전에는 `결제하기`, 시험 시간에는 `시험시작`. |
| 고위험 작업 보호 | 삭제, 환불, 성적공개, 최종제출, 파일 일괄 업로드는 재확인과 작업 이력을 둔다. |
| 공통 컴포넌트 재사용 | 탭, 필터, 테이블, 상태 배지, 모달, 파일 업로드, 날짜 선택은 모든 버전에서 같은 규칙으로 만든다. |
