# figma_pdf/2025와 ai_docs IA 비교 분석

작성일: 2026-05-21  
비교 대상:

- Figma 산출물: `figma_pdf/2025`
- IA 문서: `ai_docs/ia분석파일.docx`
- 보조 참고: `figma_pdf_2024_2025_vibecoding_비교분석.md`

## 1. 결론 요약

`ai_docs/ia분석파일.docx`는 시험 플랫폼 전체 IA를 넓게 정의한다. 수험생, 출제자, 채점자, 관리자 권한을 나누고, 시험접수, 시험응시, 출제, 채점, 결과, 자격증, 결제, 고객센터, 통계, 시스템 관리까지 운영 플랫폼 전체를 설명한다.

반면 `figma_pdf/2025`는 전체 IA를 모두 다시 그린 자료가 아니다. 2025 산출물은 기존 시험 플랫폼의 일부 관리자/전문가 화면을 유지하면서, 창작작업실·메타-T 번역·양식관리·템플릿 양식 관리자·장바구니/결제 같은 화면 패턴을 참고 자료로 제시한 묶음에 가깝다.

따라서 두 자료의 관계는 다음처럼 보는 것이 가장 자연스럽다.

| 구분 | 역할 |
|---|---|
| `ai_docs/ia분석파일.docx` | 서비스 전체의 기본 IA, 권한, 시험 운영 업무 정의 |
| `figma_pdf/2025` | 2025년에 참고할 UX/UI 패턴과 양식/템플릿/결제/일부 관리자 화면 상세 시안 |

중요하게, `figma_pdf/2025/창작작업실01.png`는 현재 기준에서 신규 핵심 기능 확정 화면이라기보다 UX/UI 참고 화면으로 보는 것이 적절하다. `Group 758531575.jpg` 역시 독립 화면이나 메뉴가 아니라, 관리자 양식/설정 요소가 작업형 참고 화면과 번역/편집 패널로 이어질 수 있음을 설명하는 자료다.

## 2. ai_docs IA의 핵심 구조

`ai_docs/ia분석파일.docx`에서 확인되는 최상위 홈 구조는 다음과 같다.

| 홈 구분 | 최종 대메뉴 |
|---|---|
| 수험생홈 | 시험관리, 자격증관리, 결제관리, 마이페이지, 고객센터 |
| 출제자홈 | 출제관리, 시험관리, 자료관리, 마이페이지, 고객센터 |
| 채점자홈 | 채점관리, 시험관리, 자료관리, 마이페이지, 고객센터 |
| 관리자홈 | 회원관리, 시험관리, 출제관리, 채점관리, 결과관리, 자격증관리, 결제관리, 고객센터, 통계관리, 시스템관리 |

관리자 서비스 구조는 다음 기능을 중심으로 정의되어 있다.

| 번호 | 대메뉴 | 주요 기능 |
|---:|---|---|
| 1 | 홈 Dashboard | 신규 회원 수, 시험 접수 현황, 결제 현황, 환불 대기, 문제 검수 대기, 문의 미답변, 바로가기 |
| 2 | 회원관리 | 사용자/출제자/채점자/관리자 조회, 승인/반려, 정지/복구, 그룹 변경, 상세보기, 일괄 처리, 엑셀 다운로드 |
| 3 | 시험관리 | 시험 차수 등록, 접수 기간, 시험 시간, 결과 발표일, 문제 검수, 문제은행, 시험 결과 관리 |
| 4 | 결제관리 | 매출, 응시료, 자격증 발급비, PG 결제, 환불 승인, 정산 보고서 |
| 5 | 고객센터 | 1:1 문의 통합 조회, 문의 분류, 답변 등록, 답변 템플릿, 처리 이력 |
| 6 | 회원정보관리 | 관리자 정보 수정, 비밀번호 변경, 접근 로그, 권한 설정, 보안 설정 |
| 7 | 통계 | 통합, 회원, 시험, 결제, 합격자, 자격증, 교재 통계 |
| 8 | 분류코드 | 시험 분류코드, 언어 코드, 등급 코드, 대/중/소분류 코드, 사용 여부 및 순서 |

ai_docs는 시험 운영 플랫폼으로서의 통제성, 추적성, 분석성, 보안성을 강조한다. 특히 관리자 화면은 단순 목록 화면이 아니라 시험 플랫폼 운영 컨트롤 타워로 설계되어야 한다고 설명한다.

## 6. ai_docs에 부족하고 2025에서 새로 보이는 영역

2025 산출물에서 새로 중요해진 기능은 ai_docs에 충분히 반영되어 있지 않다.

| 2025 신규/확장 영역 | 2025 근거 | ai_docs 상태 | 보완 필요 |
|---|---|---|---|
| 창작작업실 | `창작작업실01.png` | 없음 또는 매우 약함 | 현재는 일반 사용자용 AI 작업 대메뉴 확정보다 UX/UI 참고 자료로 분류한다. |
| 메타-T 번역 | `메타번역.png` | 시험/번역 도메인은 있으나 서비스형 번역 작업 IA는 부족 | 번역 유형, 언어, AI 엔진, 편집기, 결제 플로우를 정의해야 한다. |
| 양식관리 | `양식관리*.pdf` | 없음 | 관리자 IA에 AI 양식관리 메뉴를 추가해야 한다. |
| 템플릿 양식 관리자 | `템플릿 양식 관리자.pdf` | 없음 | 추천 서식/템플릿 운영 정책을 정의해야 한다. |
| AI 엔진 선택 | 창작작업실, 메타번역 | AI 분석 페이지 정도만 언급 | ChatGPT, Gemini, wrtn. 등 엔진 선택/결과 저장 정책 필요 |

## 7. 2025에 부족하고 ai_docs에만 있는 영역

반대로 ai_docs에는 있지만 2025 산출물에서 직접 화면으로 확인되지 않는 영역도 많다.

| ai_docs 영역 | 2025 상태 | 해석 |
|---|---|---|
| 수험생 시험접수 | 직접 화면 없음 | 2025는 수험생 시험접수 전체를 다룬 자료가 아니다. |
| 수험생 시험응시 | 직접 화면 없음 | 2024 또는 별도 자료를 기준으로 해야 한다. |
| 수험생 시험결과 | 직접 화면 없음 | 2025에서는 자격증/결제 일부만 확인된다. |
| 출제자 배정 시험 | 일부만 확인 | 프롬프트/단답형 출제 화면은 있으나 배정 목록은 부족하다. |
| 채점자 채점 화면 | 일부만 확인 | 단답형 채점 상세 PDF만 존재한다. |
| 고객센터 상세 | 상세 화면 부족 | 상단 메뉴만 확인된다. |
| 공지/FAQ | 직접 화면 없음 | ai_docs 기준으로 별도 설계 필요 |
| 결과관리 | 직접 화면 부족 | 관리자 결과 확정/공개 화면은 2025에서 명확하지 않다. |
| 시스템관리 | 직접 화면 없음 | 보안설정, 로그관리, 기본설정은 ai_docs 기준으로 별도 설계 필요 |
| 통계 세부 탭 | 직접 화면 부족 | 대시보드 PDF와 회원관리 통계 탭만 확인된다. |

즉 `figma_pdf/2025`만으로 전체 서비스를 구현하면 수험생 시험 흐름과 운영 통제 기능이 빠진다. 전체 IA는 ai_docs를 기준으로 잡고, 2025는 기반 UX/UI와 일부 화면 패턴을 보강하는 참고 자료로 써야 한다.

## 8. Group 758531575 해석 및 반영 방식

`Group 758531575.jpg`는 다음 화면 요소를 한 이미지 안에 배치하고 보라색 선으로 연결한다.

| 위치 | 내용 | 해석 |
|---|---|---|
| 좌측 상단 | 관리자/양식 설정처럼 보이는 고밀도 화면 | 카테고리, 필터, 설정, 체크박스, 세부 옵션을 관리하는 화면 |
| 우측 상단 | 작업형 입력 참고 화면 | 사용자가 파일/텍스트를 입력하고 작업을 시작하는 UX/UI 참고 화면 |
| 하단 | 원문 텍스트, AI 번역기, 비교 번역기, 에디터, 추천 단어 패널 | 입력된 작업이 실제 번역/편집 작업 공간으로 펼쳐진 상태 |
| 연결선 | 관리자 설정에서 사용자 화면과 편집 패널로 이어짐 | 설정값이 사용자 UI와 작업 패널에 반영된다는 설명 |

문서 반영 기준:

1. 별도 메뉴로 등록하지 않는다.
2. 별도 페이지 구현 대상으로 잡지 않는다.
3. `양식관리 -> 작업형 참고 화면 -> 번역/편집 패널` 연결을 설명하는 근거 자료로 사용한다.
4. 기능 요구사항에는 "관리자 설정값이 사용자 입력 화면과 작업 패널에 반영되어야 한다"는 요구로 반영한다.
5. 화면 수 산정에서는 제외한다.

| `Group 758531575.jpg` | 없음 | 기능 연결 설명 자료다. 별도 페이지가 아니라 양식관리와 창작작업실 연결 근거다. |

## 13. 최종 정리

전체 서비스를 설계할 때는 `ai_docs/ia분석파일.docx`를 기본 골격으로 삼아야 한다. 이 문서는 권한, 시험 운영, 결제, 고객센터, 통계, 보안 같은 운영 플랫폼의 기본 구조를 잘 잡고 있다.

다만 2025 Figma에는 ai_docs에 없는 신규 서비스 방향과 UX/UI 참고 패턴이 들어 있다. 핵심은 작업형 화면, 메타-T 번역, 양식관리, 템플릿관리, 결제, 전문가 검수/견적 가능성이다. 이 영역은 바로 구현 범위로 확정하기보다 검토 항목으로 반영해야 한다.

특히 `창작작업실01.png`와 `Group 758531575.jpg`는 구현 대상이라기보다 참고 자료로 다뤄야 한다. 관리자가 만든 양식과 옵션이 사용자 입력 화면에 표시되고, 그 결과가 번역/편집 패널로 이어질 수 있다는 관계를 보여주는 자료로 보는 것이 맞다. 이 해석을 기준으로 문서와 구현 범위를 잡아야 화면 수와 기능 범위가 과대 산정되지 않는다.

## 14. UX/UI 관점 비교 보완

`ai_docs`는 시험 플랫폼의 안정적인 업무 흐름을 설명하고, 2025 Figma는 작업형 UX/UI와 양식 기반 입력 화면 참고를 더 많이 보여준다. 이 둘을 합치더라도 곧바로 생산성 도구를 확정한다기보다, 시험 플랫폼 UX/UI 개선 참고 자료로 활용하는 것이 안전하다.

| 항목 | ai_docs 기준 | 2025 Figma 기준 | UX/UI 통합 방향 |
|---|---|---|---|
| 사용자 목표 | 시험 접수, 응시, 출제, 채점, 결과 확인 | AI 작업 생성, 번역, 파일 업로드, 양식 선택 | 목표가 다르므로 첫 화면과 주요 버튼을 분리한다. |
| 화면 밀도 | 업무 관리형 목록/상세 중심 | 작업실, 편집기, 설정 패널 중심 | 관리 화면은 고밀도, 작업 화면은 집중형 레이아웃으로 구분한다. |
| 카테고리 | 시험 분류 중심 | 작업 유형/양식/템플릿 분류 중심 | 카테고리 체계를 공통화하되 사용 화면에서는 목적별로 다르게 노출한다. |
| 결제 | 시험/자격증 결제 중심 | 작업 옵션, 검수, 구독, 장바구니 가능성 | 가격 산정 과정을 사용자가 이해할 수 있게 단계별로 보여준다. |
| 전문가 | 출제자/채점자 역할 중심 | 전문가 프로필과 검수 가능성 강화 | 전문가를 시험 업무자와 AI 작업 검수자로 모두 정의해야 한다. |

### 14.1 2025에서 특히 보완할 UX

| 보완점 | 이유 |
|---|---|
| 양식 미리보기 | 관리자가 만든 양식이 사용자 작업실에서 어떻게 보일지 확인해야 한다. |
| 작업 진행 상태 | AI 처리, 전문가 검수, 결제 대기, 완료 상태가 명확해야 한다. |
| 파일 업로드 피드백 | 업로드 성공/실패, 처리중, 지원 형식, 용량 제한 안내가 필요하다. |
| 결과 비교 화면 | 여러 AI 결과와 최종 편집본을 비교할 수 있어야 한다. |
| 옵션 가격 표시 | 긴급, 고급전문가, 검수 옵션이 비용에 미치는 영향을 즉시 보여줘야 한다. |

### 14.2 기존 IA에 추가해야 할 UX 문서 항목

| 문서 항목 | 정의할 내용 |
|---|---|
| 작업 생성 플로우 | 사용자가 작업 유형을 선택하고 결과를 받기까지의 단계 |
| AI 처리 상태 | 대기, 처리중, 실패, 완료, 재시도 상태 |
| 전문가 검수 플로우 | 검수 요청, 견적, 배정, 검수 완료, 반려 기준 |
| 양식 노출 규칙 | 어떤 양식이 어떤 카테고리/사용자/요금제에 보이는지 |
| 결과물 저장 정책 | 원본 파일, AI 결과, 최종 편집본, 다운로드 파일 보관 기준 |
