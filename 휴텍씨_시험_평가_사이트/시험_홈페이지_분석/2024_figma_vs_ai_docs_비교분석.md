# 2024 Figma 자료와 ai_docs IA 문서 비교 분석

작성일: 2026-05-21  
비교 대상 A: `figma_pdf/2024`  
비교 대상 B: `ai_docs/ia분석파일.docx`  
목적: 2024 Figma 산출물의 실제 화면 범위와 AI 문서의 IA/기능 정의가 얼마나 일치하는지 확인하고, 누락/과잉/보완 항목을 정리한다.

## 1. 결론 요약

두 자료는 큰 방향에서는 같은 서비스를 설명한다. 공통 핵심은 “시험 플랫폼”이며, 수험생, 출제자, 채점자, 관리자 네 역할이 존재하고, 시험 접수부터 응시, 출제, 채점, 결과, 자격증, 결제, 고객지원, 통계까지 이어지는 구조를 가진다.

다만 문서의 성격은 다르다.

| 구분 | 2024 Figma 자료 | ai_docs IA 문서 |
|---|---|---|
| 성격 | 실제 화면 시안/화면 목록 | 기능 명세와 IA 설계 문서 |
| 강점 | 실제 화면명, 팝업, 상태별 화면, 관리자 테이블, 응시 화면까지 구체적 | 역할별 메뉴 구조, 기능 목적, 상태값, 프로세스 설명이 체계적 |
| 약점 | 화면별 상세 정책 설명은 파일명만으로 제한적 | 실제 Figma에 있는 일부 운영/미디어/결제 세부 화면이 빠져 있거나 추상적 |
| 활용 방식 | 구현 화면 목록과 UI 기준으로 사용 | 요구사항, 메뉴 구조, 상태값, 권한 흐름 기준으로 사용 |

따라서 구현 기준은 `ai_docs`의 IA를 상위 요구사항으로 두고, `figma_pdf/2024`를 실제 화면 목록과 UI 패턴 기준으로 삼는 방식이 적절하다.

## 2. 상위 메뉴 비교

### 2.1 ai_docs의 상위 역할 구조

`ai_docs/ia분석파일.docx`는 홈을 다음 네 가지로 나눈다.

| 홈 | 대메뉴 |
|---|---|
| 수험생홈 | 시험관리, 자격증관리, 결제관리, 마이페이지, 고객센터 |
| 출제자홈 | 출제관리, 시험관리, 자료관리, 마이페이지, 고객센터 |
| 채점자홈 | 채점관리, 시험관리, 자료관리, 마이페이지, 고객센터 |
| 관리자홈 | 회원관리, 시험관리, 출제관리, 채점관리, 결과관리, 자격증관리, 결제관리, 고객센터, 통계관리, 시스템관리 |

### 2.2 2024 Figma의 상단 메뉴

`메뉴 구조.png`에서 확인되는 상단 메뉴는 다음과 같다.

| Figma 상단 메뉴 | 해석 |
|---|---|
| 관리자 홈 | 관리자 대시보드 |
| 회원관리 | 회원/권한/그룹 관리 |
| 시험관리 | 일정, 응시, 결과, 자격증 |
| 문제관리 | 출제, 출제내역, 문제 관련 관리 |
| 결제관리 | 정산, 결제, 카드, 자격증 정산 |
| 코드추가 | 분류/코드 관리 |
| 1:1 문의 관리 | 문의 관리 |
| 내 정보 | 계정 정보 |

### 2.3 차이점

| 항목 | ai_docs | Figma 2024 | 판단 |
|---|---|---|---|
| 결과관리 | 관리자 대메뉴로 별도 존재 | 시험관리/시험결과 화면으로 포함된 것으로 보임 | Figma는 결과를 시험관리 하위로 통합 |
| 자격증관리 | 관리자 대메뉴로 별도 존재 | 시험관리 하위 또는 수험생 자격증신청 화면으로 존재 | 구현 시 별도 메뉴 또는 시험관리 하위 중 결정 필요 |
| 시스템관리 | 관리자 대메뉴로 존재 | 코드추가, 사이트관리, 관리자관리, 알림창관리 등으로 분산 | Figma 기준으로는 운영관리/시스템관리 재분류 필요 |
| 자료관리 | 출제자/채점자 메뉴에 존재 | 명시적 파일명은 적음. STT/TTS/Youtube/영상 자료 화면은 존재 | 자료관리의 범위를 명확히 해야 함 |
| 채점관리 상단 메뉴 | ai_docs 관리자홈에 존재 | `메뉴 구조.png` 상단에는 채점관리 항목이 보임 | 일치 |

## 3. 역할별 기능 비교

### 3.1 수험생

| 기능 | ai_docs 정의 | Figma 2024 화면 | 일치 여부 |
|---|---|---|---|
| 시험 접수 | 시험 일정 확인, 시험 신청, 접수 취소, 시험 연기 신청 | `목록_시험접수.pdf`, `목록_시험접수_변경.pdf`, `목록_시험접수_취소.pdf`, `목록_시험접수_팝업.pdf` | 높음 |
| 시험 응시 | 시험 시작, 응시 유의사항 확인 | `목록_시험응시.pdf`, `응시생_시험시작.png`, `응시생_시험중.png`, `응시생_제출.png` | 높음 |
| 시험 결과 | 성적 확인, 합격 여부 확인 | `목록_시험결과.pdf` | 높음 |
| 자격증 신청 | 자격증 발급 신청, 발급 상태 확인 | `목록_자격증신청.pdf`, `수험생_자격증 신청하기.png` | 높음 |
| 결제 내역 | 시험/자격증 결제내역 확인, 환불 신청 | `목록_결제내역.pdf`, `결제*.pdf`, `포인트 결제*.pdf` | 높음 |
| 마이페이지 | 회원정보 수정, 비밀번호 변경 | `마이페이지_1_1 문의 상세.pdf`, `프로필관리.pdf`, 비밀번호 관련 PDF | 부분 일치 |
| 고객센터 | 문의 등록, 답변 확인 | `목록_1_1문의.pdf`, `문의팝업.pdf`, `마이페이지_1_1 문의 상세.pdf` | 높음 |

수험생 영역은 가장 잘 맞는다. Figma에는 ai_docs보다 시험 응시 중 화면이 더 구체적으로 존재한다. 특히 프롬프트형, 단답형, 문장나누기, 제출확인, 종료 화면은 ai_docs의 일반 설명보다 실제 구현에 더 직접적이다.

### 3.2 출제자

| 기능 | ai_docs 정의 | Figma 2024 화면 | 일치 여부 |
|---|---|---|---|
| 새 문제 출제 | 시험 조건 선택, 지문/문제/정답/해설/평가기준 입력 | `출제관리_번역상세.pdf`, `출제관리_프롬프트상세.pdf`, `출제관리_단답형상세.pdf` | 높음 |
| 출제 이어하기 | 임시저장 문항 불러오기 | `임시저장.pdf`, `저장.pdf`, `저장팝업.pdf` | 부분 일치 |
| 출제내역 확인 | 작성중/제출/반려/승인 상태 확인 | `목록_출제내역.pdf`, `시험 출제내역.png` | 높음 |
| 반려 문제 수정 | 관리자 반려 후 수정/재제출 | 명시적 반려 파일명은 적으나 상세/저장 화면으로 추정 | 부분 일치 |
| 문제 유형 | 객관식, 주관식, 서술형, 번역형, 프롬프트형, 윤리형 | 번역형, 프롬프트형, 단답형 중심 | 일부 차이 |

ai_docs는 문제 유형을 넓게 정의하지만, 2024 Figma에서 명확히 보이는 유형은 번역형, 프롬프트형, 단답형이 중심이다. 객관식, 윤리형, 서술형은 문서상 요구로 남아 있고, 실제 화면 존재 여부는 추가 확인이 필요하다.

### 3.3 채점자

| 기능 | ai_docs 정의 | Figma 2024 화면 | 일치 여부 |
|---|---|---|---|
| 채점 시작 | 배정 시험 확인, 채점 시작 | `채점자 홈_시험 채점하기.png` | 높음 |
| 개별 답안 채점 | 문제/답안/기준표 확인 후 점수 입력 | `채점자 홈_시험 채점하기_개별채점하기.png`, `상세_번역채점.pdf`, `상세_프롬프트채점.pdf` | 높음 |
| 채점 내역 | 완료, 미완료, 재검토 상태 확인 | `목록_채점내역.pdf`, `채점자 홈_채점내역.png` | 높음 |
| 피드백 | 코멘트 또는 평가 의견 입력 | `상세_번역채점_피드백.pdf`, `평가서 팝업.png` | 높음 |
| 교차 채점/블라인드 채점 | ai_docs에 필요 기능으로 제안 | Figma 파일명 기준 명확하지 않음 | 문서에만 있음 |

채점 기본 흐름은 일치한다. 다만 ai_docs가 제안한 블라인드 채점, 중복 채점, 교차 채점, 채점 기준 고정 같은 고도화 기능은 Figma 파일명만으로는 확인되지 않는다. 실제 화면 정책으로 채택할지 결정이 필요하다.

### 3.4 관리자

| 기능 | ai_docs 정의 | Figma 2024 화면 | 일치 여부 |
|---|---|---|---|
| 홈 Dashboard | 운영 요약, 신규 회원, 접수, 결제, 환불, 검수, 문의 | `관리자 홈.png` | 부분 일치 |
| 회원관리 | 사용자/출제자/채점자/관리자 조회, 승인, 정지, 그룹 변경 | `회원관리.pdf`, `회원관리_상세_전체.pdf`, `그룹관리.pdf`, `관리자관리.pdf` | 높음 |
| 시험관리 | 일정 등록, 시험 설정, 응시자 목록, 결과 관리 | `시험관리.pdf`, `시험관리_신규.pdf`, `관리자 홈.png` | 높음 |
| 문제관리 | 출제 문항 검수, 승인/반려, 문제은행 | `출제관리*.pdf`, `카테고리관리.pdf`, `구분*.pdf` | 부분 일치 |
| 채점관리 | 채점 배정, 현황, 검수 | `채점관리*.pdf`, `채점자 홈*.png` | 높음 |
| 결제관리 | 매출, 환불, 정산 보고서 | `결제관리.pdf`, `결제미승인.pdf`, `결제완료.pdf`, `정산관리.pdf` | 높음 |
| 고객센터 | 문의 답변, 공지, FAQ | `1_1 문의관리.pdf`, `게시판+FAQ.pdf`, `고객지원.pdf` | 높음 |
| 통계 | 회원/시험/결제/합격자/자격증/교재 통계 | `통계.pdf` | 부분 일치 |
| 시스템관리 | 기본설정, 보안설정, 로그관리 | `사이트관리.pdf`, `관리자관리.pdf`, `코드추가/구분*.pdf` | 부분 일치 |

관리자 영역은 범위가 매우 넓게 일치한다. 다만 ai_docs의 “관리자 서비스 구조”는 개념적으로 정돈되어 있고, Figma는 실제 화면 단위가 더 세분화되어 있다. 예를 들어 Figma에는 `사이트관리_메인배너(수정).pdf`, `알림창관리.png`, `이메일발송.pdf`, `블로그_목록.pdf`처럼 운영 실무 화면이 더 구체적으로 존재한다.

## 4. Figma에는 있지만 ai_docs에서 약한 항목

다음 항목은 Figma 2024 자료에 화면으로 존재하지만, ai_docs에서는 상대적으로 덜 강조되거나 명시가 약하다.

| 항목 | Figma 화면 | 보완 필요 |
|---|---|---|
| 알림창 관리 | `알림창관리.png` | 운영관리 하위에 팝업/알림창 관리 기능 추가 |
| 블로그 운영 | `블로그_목록.pdf`, `블로그_상세.pdf` | 고객센터 또는 콘텐츠관리 메뉴에 포함 필요 |
| 사이트 배너 수정 | `사이트관리_메인배너(수정).pdf` | 시스템관리/사이트관리 상세 기능으로 추가 |
| 이메일 발송 | `이메일발송.pdf` | 회원/시험/공지 연계 발송 기능 정의 필요 |
| 요금제/장바구니 | `요금제.pdf`, `장바구니.pdf` | 결제관리 또는 상품/요금제 관리로 분류 필요 |
| 포인트 결제 | `포인트 결제.pdf`, `포인트 결제-1.pdf` | 결제수단 정책에 포인트 추가 |
| STT/TTS/STS | `STT.pdf`, `TTS.pdf`, `STS.pdf` | 자료관리 또는 AI/미디어 기능으로 메뉴 정의 필요 |
| Youtube/영상 | `Youtube.pdf`, `영상.pdf` | 영상 기반 시험/자료 처리 기능 정의 필요 |
| 웹툰 번역 | `시험_웹툰번역(번역기선택).png` | 시험 유형에 이미지/웹툰 번역 추가 |
| 관리자 추가 버튼 | `관리자 추가 버튼들 - 미완료.pdf` | 미완료 액션 목록 정리 필요 |
| 빈 상태 | `정보없을경우.pdf` | 공통 UI 상태 정의에 추가 |

## 5. ai_docs에는 있지만 Figma에서 약한 항목

다음 항목은 ai_docs에 명확히 정의되어 있으나, 2024 Figma 파일명 기준으로는 직접 확인이 어렵다.

| 항목 | ai_docs 내용 | Figma 확인 상태 | 판단 |
|---|---|---|---|
| 자료관리 | 출제 가이드, 채점 가이드, 자료보기 | 명시적 `자료관리` 파일명 부족 | STT/TTS/Youtube/영상과 통합할지 결정 필요 |
| 교차 채점 | 중복 채점/교차 채점 | 파일명으로 확인 어려움 | 채점 정책 기능으로 별도 설계 필요 |
| 블라인드 채점 | 개인정보를 숨긴 채 채점 | 파일명으로 확인 어려움 | 채점 화면 정책으로 추가 가능 |
| 접근 로그 | 관리자 접근 로그 확인 | 파일명으로 확인 어려움 | 시스템관리 또는 관리자관리 하위로 추가 필요 |
| 보안 설정 | 권한, 보안, 접근제어 | 일부 관리자관리로 추정 | 세부 권한 모델 필요 |
| 컷오프 설정 | 합격 기준 설정과 합격자 사정 | 파일명으로 직접 확인 어려움 | 시험결과 관리 상세에 포함 필요 |
| 문제은행 | 승인 문항 등록/버전 관리 | 출제/카테고리 화면으로 추정 | 문제관리 하위 메뉴로 명확히 분리 필요 |

## 8. 구현 시 우선 반영할 차이

1. Figma의 `문제관리`와 ai_docs의 `출제관리/문제관리` 용어를 통일해야 한다.
2. Figma의 `코드추가`는 개발/운영 관점에서 `코드관리` 또는 `분류코드관리`로 이름을 바꾸는 것이 이해하기 쉽다.
3. ai_docs의 `자료관리`는 Figma의 STT/TTS/Youtube/영상 화면과 연결해서 `자료/미디어 관리`로 재정의하는 것이 좋다.
4. 자격증관리는 ai_docs처럼 별도 메뉴로 둘지, Figma처럼 시험관리 하위로 둘지 결정해야 한다.
5. 결과관리는 관리자에게는 독립 메뉴 성격이 강하지만, 수험생에게는 시험관리 하위의 시험결과로 보이는 것이 자연스럽다.
6. 결제관리에는 카드/포인트/미승인/완료/정산/요금제/장바구니까지 포함해야 한다.
7. 운영관리에는 문의뿐 아니라 게시판, FAQ, 블로그, 알림창, 이메일, 사이트 배너 관리까지 포함해야 한다.
8. ai_docs가 제안한 블라인드/교차 채점, 접근 로그, 보안 설정, 컷오프 설정은 Figma 화면만으로는 부족하므로 별도 정책 정의가 필요하다.

## 10. 최종 판단

`ai_docs/ia분석파일.docx`는 서비스가 어떤 역할과 업무 흐름을 가져야 하는지 설명하는 상위 설계 문서로 적합하다. 반면 `figma_pdf/2024`는 실제로 어떤 화면을 만들어야 하는지 알려주는 화면 목록과 UI 기준 자료로 적합하다.

두 자료를 그대로 비교하면 ai_docs는 “이상적인 IA와 기능 정의”, Figma 2024는 “실제 구현해야 할 화면 묶음”에 가깝다. 따라서 최종 요구사항은 두 자료 중 하나만 선택하지 말고, 다음 기준으로 통합하는 것이 좋다.

1. 메뉴와 권한 구조는 ai_docs를 기준으로 한다.
2. 실제 화면 목록과 구현 범위는 Figma 2024를 기준으로 한다.
3. Figma에만 있는 화면은 운영/결제/미디어/공통 UI 보완 항목으로 추가한다.
4. ai_docs에만 있는 기능은 정책 기능으로 분류하고, 실제 화면 설계가 필요한지 결정한다.
5. 개발 단계에서는 `2024_figma_사이트맵_기능정의.md`를 화면 구현 체크리스트로 사용하고, 이 비교 문서를 요구사항 갭 관리 문서로 사용한다.

## 11. UX/UI 관점 추가 비교

두 자료의 가장 큰 차이는 `ai_docs`가 업무 흐름을 설명하고, Figma가 실제 화면 밀도와 조작 방식을 보여준다는 점이다. 따라서 통합 설계에서는 IA만 맞추는 것으로는 부족하고, 각 역할이 화면에서 무엇을 먼저 판단해야 하는지까지 반영해야 한다.

| 비교 항목 | ai_docs 강점 | Figma 2024 강점 | 통합 UX/UI 기준 |
|---|---|---|---|
| 역할별 홈 | 수험생/출제자/채점자/관리자 역할 구분이 명확함 | 실제 관리자 홈의 테이블과 필터 밀도가 확인됨 | 홈은 역할별로 완전히 다른 우선순위를 가져야 한다. |
| 메뉴 구조 | 대/중/소 메뉴가 체계적임 | 실제 상단 메뉴와 사이드 메뉴가 있음 | 문서의 메뉴 구조를 기준으로 하되, 화면에서는 자주 쓰는 메뉴를 더 앞에 둔다. |
| 상태값 | 상태 정의가 풍부함 | 상태 버튼과 배지가 실제 화면에 존재함 | 상태는 배지, 버튼 활성화, 다음 행동 안내에 모두 연결한다. |
| 관리자 화면 | 운영 본부라는 방향성이 명확함 | 고밀도 테이블과 필터 구조가 구체적임 | 관리자 화면은 대시보드, 목록, 상세, 대량처리를 분리해 복잡도를 낮춘다. |
| 수험생 화면 | 접수-응시-결과-자격증 흐름이 명확함 | 응시 중 화면, 제출, 종료 화면이 구체적임 | 수험생은 메뉴 탐색보다 다음 행동 버튼 중심으로 설계한다. |
| 출제/채점 | 품질 관리 흐름을 설명함 | 번역/프롬프트/단답형 상세 화면이 있음 | 작업 중 실수를 줄이기 위해 단계형 입력과 미리보기를 둔다. |

| 우선순위 | 보완 항목 | 이유 |
|---:|---|---|
