# 3차 보완 분석 메모

## 확인한 문제

2차 생성본은 메뉴와 표는 넓게 갖췄지만, 주요 액션이 대부분 확인 팝업으로만 끝났다. 실제 서비스처럼 보여야 하는 수험생 핵심 흐름인 `시험 접수 -> 결제 -> 시험 응시 -> 결과 확인 -> 자격증 신청`이 상태 기반으로 연결되지 않았다.

## 1차에서 유지할 점

- 권한별 IA 분리: 수험생, 출제자, 채점자, 관리자 메뉴를 섞지 않는다.
- 수험생, 출제자, 채점자, 관리자 홈을 각각 독립 업무 화면으로 둔다.
- 공통 테이블, 상태 배지, 역할 전환, 관리자 위젯 구조를 유지한다.

## 이번 보완

- `source/exam_service`를 새 서비스 앱으로 분리했다.
- 2차 정적 앱의 역할별 메뉴/관리자/출제자/채점자/대량관리 구조를 가져왔다.
- 수험생 핵심 플로우는 단순 팝업 대신 상태 기반 진행 화면으로 바꿨다.
- 신청 완료 시 결제 단계가 열리고, 결제 완료 시 응시 화면이 열리며, 제출 완료 후 결과와 자격증 신청 단계가 이어진다.
- 디자인 시스템 기준에 맞춰 기본 radius는 0으로 덮고, 칩/배지만 pill 형태로 유지했다.
