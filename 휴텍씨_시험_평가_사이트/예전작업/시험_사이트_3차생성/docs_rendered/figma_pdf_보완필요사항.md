# figma_pdf 보완 필요 사항

작성일: 2026-05-29

## 1. 전제

현재 3차 생성은 `source/design_system_major/docs/시험사이트_분석/*` 문서를 기준으로 구현했다. 이 문서들은 `figma_pdf`의 전체 화면을 OCR로 완전 추출한 자료가 아니라, 파일명, 대표 이미지, 일부 확인 가능한 화면 내용을 바탕으로 정리된 자료다.

따라서 전체 IA, 역할별 메뉴, 큰 업무 흐름은 파악 가능하지만, 실제 운영 수준의 화면을 맞추려면 아래 항목은 별도 Markdown 설명 또는 화면 이미지로 보완하는 것이 필요하다.

## 2. md로 보완하면 좋은 항목

| 구분 | 보완 필요 내용 | 이유 |
| --- | --- | --- |
| 권한 정책 | 수험생, 출제자, 채점자, 관리자별 접근 가능 메뉴와 버튼 권한 | PDF 화면만으로는 권한별 숨김/읽기/수정 범위가 명확하지 않다. |
| 회원/전문가 상세 | 회원 상세에서 보여야 하는 기본정보, 활동, 결제, 문의, 자격증, 전문가 승인 항목 | `회원관리_상세_전체.pdf`의 세부 필드와 탭 우선순위가 구현 기준으로 필요하다. |
| 전문가 승인 절차 | 승인, 반려, 보류, 재심사, 증빙 첨부, 관리자 메모 처리 규칙 | 버튼은 보이지만 상태 전이와 필수 입력값은 PDF만으로 확정하기 어렵다. |
| 시험 등록 정책 | 시험명, 회차, 등급, 시험유형, 일정, 접수기간, 응시료, 환불 규칙 | 시험관리 신규 화면의 입력 필드 검증 조건이 필요하다. |
| 출제 기준 | 번역형, 프롬프트형, 단답형 문항별 필수 입력값, 배점, 평가 요소 | 화면 구조는 보이나 실제 채점 기준과 저장 조건은 문서화가 필요하다. |
| 채점/평가서 | 평가서 항목명, 점수 배점, 코멘트 필수 여부, 채점완료 이후 수정 가능 여부 | 채점자 화면에서 평가서가 핵심인데 PDF만으로 최종 양식 확정이 어렵다. |
| 결제/정산/환불 | 결제 상태, 정산 상태, 환불 가능 조건, 포인트/구독/견적 요청 처리 | 화면명은 있으나 업무 규칙이 없으면 더미 UI에 머문다. |
| 자격증 발급 | 합격 조건, 발급대기/발급완료/재발급 상태, 신청서 필드 | 자격증 상세 화면과 실제 운영 정책 연결이 필요하다. |
| 양식/템플릿 | 양식 종류, 템플릿 필드, 사용자 입력 방식, 관리자 승인 방식 | 2025 자료에 화면은 있으나 실제 양식 스키마가 필요하다. |
| 대량관리 | 엑셀 업로드 컬럼, 다운로드 양식, 실패 사유, 처리 결과 메시지 | vibecoding식 대량 테이블은 컬럼 정의서가 없으면 재현이 어렵다. |
| 알림/로그 | 문자, 이메일, 카카오, 관리자 처리 로그의 발송 시점과 문구 | 화면 버튼만으로 알림 트리거가 확정되지 않는다. |
| 빈 상태/오류 상태 | 데이터 없음, 권한 없음, 저장 실패, 업로드 실패, 결제 실패 문구 | PDF에 없는 상태까지 구현하려면 운영 문구 기준이 필요하다. |

## 3. 이미지로 보완하면 좋은 항목

| 구분 | 필요한 이미지 | 이유 |
| --- | --- | --- |
| 관리자 회원관리 | 목록, 상세 펼침, 상세 페이지, 승인/반려 팝업 캡처 | 테이블 밀도, 컬럼 순서, 상세 정보 배치가 중요하다. |
| 채점자 채점 화면 | 개별 답안 채점, 평가서, 점수 입력, 채점완료 상태 캡처 | 평가서가 화면에서 얼마나 크게 보여야 하는지 이미지 기준이 필요하다. |
| 출제자 문제 작성 | 프롬프트형 설정, 프롬프트 확인, 단답형 상세 캡처 | 문제 탭, 평가 요소, 답변 예시의 실제 배치 확인이 필요하다. |
| 시험 응시 화면 | 시험 안내, 문제 풀이, 타이머, 제출 확인, 종료 상태 캡처 | 수험생 UX는 실수 방지가 중요해 화면 기준이 필요하다. |
| 결제/장바구니 | 장바구니, 결제, 포인트 결제, 구독 신청, 견적 요청 캡처 | 결제 금액 요약과 CTA 위치가 중요하다. |
| 자격증 관리 | 자격증 신청, 관리자 상세, 발급완료, 재발급 화면 캡처 | 증명서/자격증은 문서형 레이아웃 확인이 필요하다. |
| 양식/템플릿 관리 | 양식 목록, 신규 등록, 상세, 프롬프트 첫 화면, 템플릿 관리자 캡처 | 2025 신규 기능이라 PDF 해석보다 이미지 기준이 더 안전하다. |
| 대량관리 | 필터 전체 펼침, 가로 스크롤 테이블, 업로드 성공/실패 캡처 | 컬럼이 많아 텍스트 설명만으로는 UI 밀도를 맞추기 어렵다. |
| 반응형 화면 | 모바일 또는 좁은 화면에서 주요 업무 화면 캡처 | 현재 자료는 데스크톱 중심이라 모바일 기준이 부족하다. |

## 4. 우선 보완 순서

1. 관리자 `회원/전문가 관리` 상세 정보와 승인/반려 절차
2. 채점자 `개별 답안 채점`과 `평가서` 최종 레이아웃
3. 출제자 `프롬프트형/단답형 출제` 입력 필드와 평가 기준
4. 수험생 `시험 응시` 화면의 타이머, 저장, 제출 흐름
5. 결제, 정산, 환불 상태 전이
6. vibecoding 대량관리 엑셀 컬럼과 업로드 실패 처리
7. 양식/템플릿 관리의 필드 구조

## 5. 전달받으면 좋은 파일 형태

| 형태 | 권장 내용 |
| --- | --- |
| Markdown | 업무 규칙, 상태 전이, 필드 정의, 버튼 동작, 오류 문구 |
| PNG/JPG | 화면 전체 캡처, 팝업 캡처, 펼침/접힘 상태, hover나 선택 상태 |
| CSV/XLSX | 관리자 대량관리 컬럼, 시험/회원/결제/정산 샘플 데이터 |
| PDF 원본 | 화면 전체가 보이는 고해상도 원본. 가능하면 텍스트 선택 가능한 PDF |

## 6. 현재 구현에서 임시 판단한 부분

| 화면 | 임시 판단 |
| --- | --- |
| 관리자 사이드바 | 해시 링크를 탭 전환으로 연결했다. 실제 IA 확정 시 메뉴명과 순서는 조정 가능하다. |
| 회원/전문가 상세 | 현재는 행 하단 펼침 방식으로 기본정보, 권한, 상태, 메모를 보여준다. 실제 상세 필드가 오면 교체해야 한다. |
| 상단 권한 표시 | 현재 역할을 색상으로 표시한다. 색상 체계는 최종 디자인 시스템 기준으로 조정 가능하다. |
| 채점자 평가서 | 현재는 5:5 레이아웃과 총점/항목/요약 구조로 임시 구현했다. 최종 평가서 양식이 필요하다. |
| 대량관리 | 더미 데이터 기반 고밀도 테이블이다. 실제 엑셀 컬럼 정의가 필요하다. |
