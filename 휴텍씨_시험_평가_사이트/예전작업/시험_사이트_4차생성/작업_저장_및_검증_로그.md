# 작업 저장 및 검증 로그

## 현재 저장 파일

- `index.html`: 통합 정적 앱 화면
- `app.js`: 탭, 역할 메뉴, 필터, 모달, 토스트, 샘플 DB 렌더링
- `styles.css`: `design_system_major` 토큰 alias, 4단 그리드, Chip, 상태 pill, 고밀도 테이블
- `verify.js`: 반복 검증 스크립트

## 반복 작업 반영 요약

- 목차/역할별 메뉴 누락 보강
- 인증, 수험생, 출제자, 채점자, 관리자 운영 화면 추가
- 회원 상세, 마이페이지, 자격증 상세, 카테고리/코드 관리 독립 화면 추가
- 데이터 객체, 공통 컴포넌트, 업로드 분류, 라우트/정책, 완료 체크리스트 화면 추가
- `design_system_major` 기준의 DS 토큰, `filter-chip`, `state-pill`, `btn--*`, `aria-selected`, `aria-pressed`, `aria-live` 반영
- DB의 `변환`, `언어쌍` 등 고정값을 `chip fixed`로 렌더링

## 검증 명령

```bash
node verify.js
```

## 현재 검증 기준

- `app.js` 문법 확인
- 선언된 모든 `views`가 `index.html`의 `view-*` 섹션과 연결되는지 확인
- `querySelector("#...")`로 참조하는 정적 id가 HTML에 존재하는지 확인
- `design_system_major` 토큰/클래스/접근성 키워드 포함 확인
- 라우트/정책, 상세 화면, 완료 체크리스트 화면 존재 확인
- DB 샘플 15행 이상 존재 확인
- DB `변환`, `언어쌍` 컬럼과 Chip 렌더링 확인

## 최근 검증 결과

```text
Verification passed
views=23
staticIds=22
sampleRows=15
```

`staticIds` 검사는 `querySelector`와 `querySelectorAll`의 `"#id"` 및 `` `#id` `` 형태를 모두 잡도록 보정했다.

## 이어서 볼 지점

- 실제 브라우저에서 레일 메뉴가 과도하게 길어질 때 스크롤/터치 조작이 자연스러운지 확인
- 브라우저 렌더링 도구가 설치되면 주요 화면 스크린샷 검증
- 큰 화면 기준에서 `회원관리` 가로 테이블과 `완료 체크리스트` 표의 가독성 확인
