# 휴텍씨 디자인 시스템

기준 파일: `index.html`, `layout.css`, `common.js`

이 문서는 현재 `index.html`에 구현된 디자인 시스템 화면을 기준으로, 이후 시험/번역/서비스 홈페이지를 만들 때 반복 적용할 공통 규칙을 정리한 문서입니다.

## 1. 시스템 원칙

### 1.1 공통 틀 우선

홈페이지와 업무 화면은 개별 페이지를 바로 만들기보다 다음 공통 틀을 먼저 고정합니다.

- Header: 로고, 주요 메뉴, 사용자/알림 영역
- Sidebar 또는 Rail: 업무별 보조 탐색
- Content: 검색, 필터, 목록, 상세, 작성 영역
- Footer: 약관, 개인정보, SNS, 저작권

개별 홈페이지는 이 틀 위에 서비스 소개, 기능, 실적, 전문가 신청, 견적 의뢰, FAQ, 회사 소개를 조합합니다.

### 1.2 공통과 개별 분리

공통 UI와 업무 콘텐츠를 분리합니다.

- 공통 UI: 버튼, 입력, 탭, 칩, 상태 pill, 테이블, 토스트, 모달, 페이지네이션
- 개별 콘텐츠: 시험 종목, 번역 분야, 고전 번역 설명, 마케팅 문구, 신청서 항목

개별 홈페이지를 만들 때는 색상/문구/이미지/섹션 순서를 바꾸더라도 공통 UI class와 토큰은 유지합니다.

### 1.3 홈페이지 목차 원칙

첫 화면은 회사 소개보다 서비스 소개를 먼저 보여줍니다.

권장 순서:

1. 서비스 소개 / Hero
2. 기능, 분야, 현황, 기술 설명
3. 실적
4. 전문가 신청
5. 견적 의뢰
6. FAQ, 상담, 신청 흐름
7. 회사 소개

고전 번역처럼 독립 브랜드 성격이 강한 홈페이지는 기존 번역 사이트 틀을 재사용하되, 별도 색상 구조와 현황/기술 설명을 강화합니다.

| 분류 | 값 | 사용처 |

## 3. 접근성 기준

### 3.1 명도 대비

버튼, 칩, 상태 pill, badge, tag를 만들 때는 텍스트와 배경 또는 주요 경계의 명도 대비를 최소 `3:1` 이상으로 유지합니다.

현재 시스템에서 지켜야 할 조합:

- 연한 파란 배경 위 텍스트: `--ds-color-brand-primary-strong` 사용
- 컬러 버튼 위 텍스트: `--ds-color-text-inverse` 사용
- 비활성 pill: `#f2f2f2` 배경 위 `#555555` 사용
- 성공/경고/오류 텍스트: 일반 accent 색을 그대로 쓰지 말고 `*-strong` 토큰 사용

주의:

- `#0088ff` 텍스트를 `#d7e7fc` 위에 올리는 조합은 약하므로 피합니다.
- `#51b117`, `#f7b22c`를 흰 배경 위 작은 텍스트로 쓰지 않습니다. 아이콘/면 색상으로 쓰거나 strong 토큰을 함께 씁니다.
- 색상만으로 상태를 전달하지 말고 텍스트, 아이콘, `aria-pressed`, `aria-selected`, `role="switch"` 같은 상태 정보도 함께 제공합니다.

### 3.2 키보드와 포커스

- 버튼, 링크, 입력, 탭은 `:focus-visible` 스타일을 유지합니다.
- 탭은 `role="tablist"`, `role="tab"`, `aria-selected`를 사용합니다.
- 토글은 `role="switch"` 또는 `aria-pressed`를 사용합니다.
- 토스트 영역은 `aria-live="polite"`와 `aria-atomic="true"`를 사용합니다.
- 모달은 `role="dialog"`, `aria-modal="true"`, 제목 연결 `aria-labelledby`를 사용합니다.

## 4. 컴포넌트 규칙

### 4.1 Button

기본 class는 `btn`입니다. 변형은 `btn--primary`, `btn--outline`, `btn--neutral`, `btn--secondary`, `btn--danger`를 사용합니다.

크기:

- `btn--sm`: 24px
- `btn--md`: 32px
- `btn--lg`: 40px

기존 호환을 위해 `small`, `large`, `outline`, `neutral`, `secondary`, `danger` 별칭도 유지하지만 새 페이지에서는 `btn--*` 형식을 우선 사용합니다.

### 4.2 Link

본문 이동이나 테이블 내 강조 값은 버튼 대신 `text-link`를 사용합니다.

- 기본 링크: `text-link`
- 보조 링크: `text-link muted-link`
- 위험 링크: `text-link danger-link`

### 4.3 Tab

탭은 역할에 따라 class를 선택합니다.

- `tabs--underline`: 페이지 상단 주요 구분
- `tabs--box`: 상세 정보 패널
- `tabs--pill`: 상태/카테고리 필터
- `tabs--segmented`: 목록/캘린더/요약 같은 보기 전환

선택 상태는 `.active`와 `aria-selected="true"`를 함께 맞춥니다.

### 4.4 Chip, Tag, Status Pill

필터와 상태 표시는 다음 class를 사용합니다.

- 필터 칩: `filter-chip`
- 활성 필터 태그: `filter-tag`
- 규칙 필터 칩: `chip-button`
- 상태 표시: `state-pill`
- 비활성/검토중 상태: `state-pill off`
- 규칙 라벨: `rule-tag`

접근성 규칙:

- 활성 칩은 `aria-pressed="true"`를 사용합니다.
- 연한 배경 위 텍스트는 strong 토큰을 사용합니다.
- 상태는 색상만으로 구분하지 않고 "완료", "검토중", "고정", "준고정"처럼 텍스트를 같이 표시합니다.

### 4.5 Form

입력 기본 class는 `field`입니다.

- 기본 높이: 32px
- 라벨: `label`
- 상세 입력: `textarea.field`
- 도움말/오류 문구가 필요한 경우 `field-help`, `field-error` 패턴을 사용합니다.

### 4.6 Table

시험 목록은 `exam-table`을 사용합니다.

- 테이블은 `table-scroll` 안에 넣어 가로 스크롤을 허용합니다.
- 헤더는 12px 보조 텍스트, 본문은 14px 기준입니다.
- 목록 폭이 넓으면 모바일에서 억지로 줄이지 않고 가로 스크롤을 허용합니다.

### 4.7 Feedback

토스트:

- 기본 class: `toast`
- 상태 class: `success`, `warning`, `danger`, `info`
- 고정 viewport: `toast-viewport`

모달:

- 배경: `modal-backdrop`
- fixed 모달: `modal-backdrop is-fixed`
- 본문: `modal`
- 액션: `modal-actions`

## 5. 레이아웃 패턴

### 5.1 기본 페이지

- `shell`: 전체 문서 구조
- `nav`: 디자인 시스템 문서 목차
- `main`: 본문
- `section`: 문서 또는 화면 단위
- `panel`: 경계가 있는 기본 패널
- `grid two`, `grid three`, `grid four`: 반응형 그리드

### 5.2 서비스 홈페이지

홈페이지는 과한 장식보다 실제 사용자가 바로 이해할 수 있는 정보 구조를 우선합니다.

권장:

- Hero는 서비스명/서비스 소개를 첫 신호로 보여줍니다.
- CTA는 "전문가 신청", "견적 의뢰", "상담하기"처럼 실제 행동 문구로 씁니다.
- 카드가 필요한 경우 반복 항목에만 사용합니다.
- 검색/필터가 있는 화면은 인라인 칩과 활성 태그를 함께 제공합니다.

지양:

- 회사 소개를 첫 화면의 핵심으로 두는 구성
- 상태가 색상만으로 구분되는 칩/배지
- 좁은 입력칸과 한 화면 과밀 배치
- 동일 역할의 버튼 class를 페이지마다 새로 만드는 방식

## 6. 새 홈페이지 제작 체크리스트

- 공통 `layout.css` 토큰을 먼저 연결했는가?
- Header, Content, Footer 구조가 유지되는가?
- 서비스 소개가 회사 소개보다 먼저 나오는가?
- 실적, 전문가 신청, 견적 의뢰가 빠지지 않았는가?
- 버튼은 `btn btn--variant btn--size` 조합을 쓰는가?
- 칩/상태/badge/tag의 대비가 최소 `3:1` 이상인가?
- 선택 상태에 `aria-pressed` 또는 `aria-selected`가 있는가?
- 키보드 포커스가 보이는가?
- 모바일에서 테이블/필터/헤더가 깨지지 않는가?
- 색상 직접 입력보다 토큰 사용을 우선했는가?
