개요

Figma의 25년 양화 작업 페이지에서 확인한 목록, 에디터, 결제, 관리자, 팝업, 토스트 패턴을 HTML 퍼블리싱 기준으로 정리했습니다.

기본 화면 폭

1440px, 1920px 데스크톱 화면이 공존합니다.

기본 폰트

Noto Sans KR Regular/Bold, 로고는 Pretendard Black 계열입니다.

기본 UI 밀도

버튼/입력 32px, 테이블 행 60px, GNB/Footer 60px입니다.

Rules 분석

사내 Studio의 Rules 페이지에서 확인한 업무 설계 및 홈페이지 적용 원칙을 디자인 시스템의 구조 기준으로 연결했습니다.

필수 DS-01

공통 틀 우선 제작

세부 페이지나 문서를 각각 따로 만들기보다 먼저 하나의 공통 틀을 만들고 이후 여러 업무에 반복 적용한다.

디자인 시스템 반영: Header, Sidebar, Content, Footer를 공통 레이아웃으로 정의하고 체크박스·토글·보기 전환·미리보기를 그 안에 배치합니다.
필수 DS-02

확장성 중시

레이아웃 정리에서 가장 중요한 기준은 확장성으로, 이후 다른 업무에도 쉽게 붙여 쓸 수 있어야 한다.

적용: semantic CSS 토큰과 상태별 컴포넌트 class를 재사용합니다.
필수 DS-03

공통과 개별 분리

전 부서가 함께 쓰는 공통 항목과 부서별·분야별 개별 항목을 분리해서 설계한다.

적용: 공통 UI 패턴과 시험/관리 업무 콘텐츠를 분리하여 조합합니다.
필수 DS-04

번호 체계 부여

규정이나 페이지는 번호를 붙여 버전과 위치를 쉽게 구분할 수 있게 한다.

적용: 규칙 카드와 QA 항목에 DS/WEB 식별자를 노출합니다.
필수 WEB-01

화면/상태 검증

바뀌는 화면·문구·버튼 위치를 표시하고, 입력값과 관리자 출력값을 맞추며, 실서버 전 링크·버튼을 확인합니다.

적용: 아래 체크리스트와 미리보기 컴포넌트에서 반영 상태를 점검합니다.

전체 규정 체계

Rules의 업무 설계 대분류와 세부 항목 전체를 펼쳐 확인합니다. 각 항목은 디자인 시스템 화면을 설계할 때 유지할 기준입니다.

01업무 설계의 기본 원칙

프레임 우선 설계

공통 틀 우선 제작
세부 페이지나 문서를 각각 따로 만들기보다 먼저 하나의 공통 틀을 만들고 이후 여러 업무에 반복 적용한다.
찍어내기 구조 지향
기본 틀만 완성되면 이후 커리큘럼, 교재, 시험, 랜딩페이지 등은 같은 구조로 확장 생산한다.
매뉴얼 선행 정리
시간이 부족할수록 실제 제작보다 먼저 매뉴얼과 규정부터 정리하는 것이 우선이다.

구조 설계 기준

확장성 중시
레이아웃 정리에서 가장 중요한 기준은 확장성으로, 이후 다른 업무에도 쉽게 붙여 쓸 수 있어야 한다.
공통과 개별 분리
전 부서가 함께 쓰는 공통 항목과 부서별·분야별 개별 항목을 분리해서 설계한다.
번호 체계 부여
규정이나 페이지는 3-1, 3-2처럼 번호를 붙여 버전과 위치를 쉽게 구분할 수 있게 한다.
02분류 기준 및 규정 체계

기본 분류 축

대·중·소 분류
업무와 자료는 대분류-중분류-소분류 구조로 단계적으로 정리한다.
급수 구분
시험, 교육, 콘텐츠는 급수 기준과 연결해 관리할 수 있도록 본다.
분야별 구분
분야에 따라 필요한 항목과 구조가 다르므로 분야별 차이를 별도로 반영한다.

규정 구분

고정 규정
반드시 유지되어야 하는 핵심 항목으로, 공통 틀의 중심이 되는 규정이다.
준고정 규정
기본 구조는 유지하되 업무 성격에 따라 일부 변형 가능한 항목이다.
선택 규정
필요한 경우에만 추가하거나 제외할 수 있는 가변 항목이다.

분야별 차이 반영

분야별 예외 반영
법률, 고전 번역, 마케팅 등 분야마다 꼭 들어가야 하는 항목이 달라질 수 있으므로 분야별 차이를 따로 관리한다.
공통 틀 위 개별 확장
기본 틀은 같아도, 각 분야에서 필요한 세부 입력 항목은 따로 덧붙여야 한다.
03적용 대상 업무

교육 및 시험 제작

커리큘럼 설계
수업 구조를 만들고 필요한 교재 및 시험 요소를 연결하는 업무이다.
교재 제작
커리큘럼에 맞는 교재 제작 항목을 별도로 관리해야 한다.
문제은행 구축
시험 문제와 문제은행은 별도 구조와 입력 항목이 필요한 영역이다.

홈페이지 및 랜딩페이지 제작

랜딩페이지 설계
연도별, 주제별, 뉴스형 등 목적에 따라 다른 랜딩페이지 구조가 필요하다.
히어로 섹션 정의
랜딩페이지에서는 설명, 히어로 섹션, 기능, 태그 여부 등을 구조적으로 정리해야 한다.
홈페이지별 독립 구성
TESOL, 번역, 시험 등은 각각 독립 홈페이지 구조를 가질 수 있다.

마케팅 제작

마케팅 구조화
마케팅도 랜딩페이지 기반으로 헤드라인, 핵심 키워드, 대상자 설정 등 별도 구조가 필요하다.
타깃 분류
전문가 여부, 연령대, 학습 상태, 문서 유형 등 타깃 조건을 나눠야 한다.
04화면 및 기능 설계

입력 및 선택 기능

체크박스 기능
사용자가 필요한 항목을 선택해 고정·준고정·선택 구조를 빠르게 조합할 수 있게 한다.
활성/비활성 토글
동그라미 형태 등의 토글을 통해 활성/비활성 상태를 표시한다.
텍스트 입력 박스
네모박스 형태로 제목과 내용을 직접 입력할 수 있어야 한다.
버튼 바 구성
고정·준고정·선택, 레이아웃 유형 등을 빠르게 바꿀 수 있는 버튼 바가 필요하다.

보기 및 관리 기능

대시보드 보기
전체 현황을 요약해서 보는 화면이 필요하다.
리스트 보기
한 줄씩 펼쳐 보거나 목록으로 확인하는 방식이 필요하다.
미리보기 기능
일부 업무에서는 작성 전후 미리보기 창이 필요할 수 있다.
작성·추가·수정 기능
실제 내용을 입력하고, 이후 추가·수정할 수 있어야 한다.
필터 기능
특정 사람이나 특정 조건만 보이도록 필터 버튼이 필요하다.

레이아웃 유형

장문형 레이아웃
설명이 길게 들어가는 구조를 지원한다.
질문·단답형 레이아웃
질문과 짧은 답변 중심의 구조를 지원한다.
4지선다형 레이아웃
시험처럼 보기 선택 구조가 필요한 경우를 지원한다.
단일 선택형 레이아웃
여러 개 중 하나만 고르는 형태의 기능을 지원한다.

탐색 및 확장 기능

펼치기 기능 선호
사용자는 항목을 펼쳐 보며 구조를 이해하는 방식을 선호한다.
대·중·소 확장 기능
대분류를 열면 중분류가 나오고, 중분류를 열면 소분류가 보이는 확장형 구조가 필요하다.
한 화면 모드 전환
한 화면에서 대시보드형, 리스트형 등으로 모드를 바꾸는 방식도 허용된다.
05작성 및 운영 방식

입력 흐름

1차 선택 후 작성
먼저 필요한 항목을 체크한 뒤, 해당 항목이 펼쳐져서 내용을 입력하는 흐름을 만든다.
엑셀 누적 정리
기존에 엑셀에 정리한 내용을 새 구조 안으로 옮겨와 누적 관리한다.
요약 대시보드화
작성된 내용을 다시 요약해 대시보드에서 볼 수 있게 정리한다.

팀별 활용

가이드라인 배포
마케팅팀 등 각 팀에는 이 구조를 기반으로 한 가이드라인을 제공한다.
회사 방침 내재화
면접, 교육, 제작물 작성 등에서도 고정·준고정·선택 규정을 회사 방침처럼 활용한다.

프롬프트 관리

프롬프트 기록화
어떤 프롬프트를 넣었는지, 1차 결과와 2차 수정 결과가 어땠는지 기록해야 한다.
재사용 가능한 노하우화
프롬프트 기록은 개인 감이 아니라 재사용 가능한 회사 업무 노하우가 된다.
06UI/UX 선호 방향

사용성 중심 설계

좁고 빡센 틀 지양
입력칸이 좁고 사용자가 쓰기 싫어지는 구조는 피한다.
한 줄형 선호
한 줄씩 펼쳐서 바로 붙여 넣고 쓰는 구조를 선호한다.
과한 화려함 지양
스케줄·달력 등은 지나치게 화려하기보다 실용적으로 보여야 한다.
상태 가시성 강조
고정·준고정 상태가 화면에서 바로 보이면 좋다.

배치 원칙

페이지 분리 허용
한 화면에 다 우겨 넣기 어렵다면 페이지를 분리하되 연결 구조는 유지한다.
3단·4열 활용
기본 구조는 3단 또는 4열 등으로 배치하되, 상황에 따라 리스트를 아래로 빼는 것도 허용한다.
07홈페이지 구성 원칙

기본 목차 원칙

서비스 소개 우선
홈페이지 첫 화면에서는 회사 소개보다 서비스 소개가 먼저 와야 한다.
회사 소개 후순위
회사 소개는 보통 하단부에 배치하는 원칙을 둔다.
필수 항목 유지
실적, 전문가 신청, 견적 의뢰와 같은 항목은 빠지지 않도록 반영한다.

고전 번역 사이트 제작 원칙

기존 틀 재활용
기존 번역 전체 사이트 구조를 바탕으로 고전 번역용으로 변형하는 방식으로 접근한다.
내용 교체 방식
전문가 신청, 번역 신청 등 기존 항목은 유지하되 내용을 "고전"에 맞게 바꿔 적용한다.
현황·기술 강조
고전 번역 사이트에는 번역 현황이나 기술 설명처럼 다른 사이트보다 더 강조할 내용이 있다.
독립 사이트 운영
고전 번역 사이트는 기존 사이트의 하위가 아니라 별도 색상과 별도 구조를 가진 독립 사이트로 본다.

제작 절차

자료 조사 선행
홈페이지 제작 전, 먼저 해당 분야 자료를 충분히 조사해 표로 정리한다.
목차 설계 후 검토
조사 결과를 기반으로 홈페이지 목차를 만든 뒤, 빠진 요소를 다시 추가해 재구성한다.
파일 2종 활용
내용 자료와 스타일 참고 자료를 함께 넣고, 이를 바탕으로 새 홈페이지를 제작한다.

function showToast(message, type = "success", position = "top-right") {
  const viewport = document.querySelector("[data-toast-viewport]");
  viewport.dataset.toastPosition = position;

  const toast = document.createElement("div");
  toast.className = `toast ${type}`;
  toast.innerHTML = `<i class="toast-dot"></i><span>${message}</span>`;
  viewport.prepend(toast);
  window.setTimeout(() => toast.remove(), 3000);
}
