# 마크다운 에디터 고도화 프롬프트

## 작업 프롬프트

브라우저에서 바로 실행되는 마크다운 에디터와 Word 변환기를 고도화한다.

현재 앱 상태:
- 단일 파일 앱: `markdown-to-word.html`
- HTML, CSS, JavaScript만 사용
- 빌드 과정 없음
- 백엔드 없음
- 외부 패키지 의존성 없음
- 마크다운 파일 드래그 앤 드롭 지원
- 여러 파일 선택 지원
- 변환 전 미리보기 지원
- 마크다운 블록을 `.docx` 파일로 변환
- 여러 문서를 변환할 때 ZIP 다운로드 지원

핵심 목표:
- 단순한 Markdown to Word 변환기를 실무용 마크다운 에디터로 확장한다.
- 작성, 미리보기, Word 내보내기, 문서 형식 조정이 한 화면에서 자연스럽게 이어지도록 만든다.

작업 원칙:
- 특별한 이유가 생기기 전까지 브라우저 전용 앱 구조를 유지한다.
- 기존 Word 변환 기능은 깨지지 않게 보존한다.
- 기능은 한 번에 크게 갈아엎지 말고 검증 가능한 단위로 추가한다.
- UI 문구는 한국어로 짧고 명확하게 작성한다.
- 장식적인 화면보다 문서 작성과 검토 효율을 우선한다.
- 에디터 흐름과 내보내기 요구사항이 확정되기 전에는 대규모 구조 변경을 피한다.

우선 고도화 방향:
- 파일 업로드뿐 아니라 직접 마크다운을 작성하고 수정할 수 있게 한다.
- 입력한 내용과 미리보기가 실시간으로 동기화되게 한다.
- 현재 편집 중인 문서를 바로 Word로 내보낼 수 있게 한다.
- 제목, 저장 파일명, 글꼴, 여백, 표 스타일 같은 문서 설정을 추가한다.
- 브라우저 로컬 임시 저장 기능을 추가한다.
- 모바일에서도 파일 선택, 편집, 미리보기, 내보내기 흐름이 무너지지 않게 한다.

다음 대화 흐름:
1. 첫 화면을 어떤 에디터 구조로 만들지 정한다.
2. 파일 업로드와 직접 편집을 한 화면에 둘지, 탭으로 나눌지 정한다.
3. 최소 기능부터 추가한다: 입력창, 실시간 미리보기, 임시 저장, 현재 문서 Word 내보내기.
4. 기존 `.docx` 변환 기능이 그대로 동작하는지 확인한다.
5. Word 문서 품질을 제목, 목록, 표, 코드블록, 인용문, 링크 순서로 다듬는다.

결정할 사항:
- 앱 이름을 `Markdown to Word 변환기`로 유지할지, `마크다운 에디터`로 바꿀지
- Word 결과물을 업무 보고서 스타일로 만들지, 읽기 좋은 기본 문서 스타일로 만들지
- 여러 파일 일괄 변환을 핵심 기능으로 둘지, 보조 기능으로 분리할지
