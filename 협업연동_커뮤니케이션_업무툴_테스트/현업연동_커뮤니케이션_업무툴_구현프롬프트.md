# 현업연동 커뮤니케이션 업무툴 구현 프롬프트

## 목적

현업 운영자가 업무를 빠르게 생성, 검색, 비교, 상세 확인, 댓글 기록, 첨부/히스토리 추적할 수 있는 BO 관리자형 업무 협업툴을 만든다.

사용자용 랜딩페이지가 아니라 실무자가 매일 쓰는 고밀도 업무 화면이어야 한다.

## 참고 기준

- 디자인 기준은 반드시 `../design_system_major`의 BO 관리자 디자인 문서만 따른다.
  - `docs/재사용_디자인_프롬프트/관리자/02_BO_테이블레이아웃_기준.md`
  - `docs/재사용_디자인_프롬프트/관리자/04_BO_관리자용_레이아웃_기준.md`
  - `docs/재사용_디자인_프롬프트/관리자/components/인라인_Drawer_기준.md`
  - `docs/재사용_디자인_프롬프트/관리자/components/테이블선택_Drawer_상세패널_기준.md`
  - `docs/재사용_디자인_프롬프트/관리자/components/상태_유형_칩_기준.md`
- 현재 폴더의 `index2.html`, `index.html`, `../notations_tools/public/index.html`은 데이터 항목, 화면 흐름, 기능 동작만 참고한다.
- 샘플 HTML의 색상, 테두리, 여백, 카드 스타일, 이모지 중심 UI, 임의 클래스 구조는 디자인 기준으로 사용하지 않는다.

## 실행 프롬프트

```text
너는 BO 관리자형 업무 협업툴을 설계하고 구현하는 시니어 프론트엔드 엔지니어다.

현재 폴더에 `현업연동_커뮤니케이션_업무툴`을 만든다. 기존 테스트 서비스와 디자인시스템을 조합해서, 현업 업무 생성, 목록 조회, 상세 커뮤니케이션, 첨부/히스토리 추적이 가능한 단일 HTML 기반 프로토타입을 구현한다.

참고 기준:
- 디자인 기준은 반드시 `../design_system_major`의 BO 관리자 디자인 문서만 따른다.
  - `docs/재사용_디자인_프롬프트/관리자/02_BO_테이블레이아웃_기준.md`
  - `docs/재사용_디자인_프롬프트/관리자/04_BO_관리자용_레이아웃_기준.md`
  - `docs/재사용_디자인_프롬프트/관리자/components/인라인_Drawer_기준.md`
  - `docs/재사용_디자인_프롬프트/관리자/components/테이블선택_Drawer_상세패널_기준.md`
  - `docs/재사용_디자인_프롬프트/관리자/components/상태_유형_칩_기준.md`
- 현재 폴더의 `index2.html`, `index.html`, `../notations_tools/public/index.html`은 데이터 항목, 화면 흐름, 기능 동작만 참고한다.
- 샘플 HTML의 색상, 테두리, 여백, 카드 스타일, 이모지 중심 UI, 임의 클래스 구조는 디자인 기준으로 사용하지 않는다.

목표 화면:
- 사용자용 랜딩페이지가 아니라, 현업 운영자가 업무를 빠르게 검색, 비교, 생성, 상세 확인, 댓글 기록할 수 있는 BO 업무툴이다.
- 전체 레이아웃은 Sidebar + Topbar + Page Head + Dense Filter + Summary Strip + 업무 테이블 + 우측 Inline Detail Drawer 구조로 만든다.
- 목록과 상세가 동시에 보이는 2-pane 또는 3-pane 구조를 유지한다.
- 업무 상세는 모달이 아니라 우측 inline drawer/pane로 열리게 한다.

핵심 기능:
1. 업무 목록
   - `index2.html`의 마스터 그리드 느낌을 유지한다.
   - 컬럼은 다음 정보를 포함한다.
     - 선택
     - 업무번호
     - 시작일
     - 생성인
     - 대/중/소 계층 분류
     - 대표 업무 제목
     - 중요/특이사항
     - 세부내용 요약
     - main 마감일
     - HOST 마감일
     - main 담당자
     - sub 협업자
     - 공유자료
     - 상태/우선순위
   - 관련 정보는 BO 테이블 기준에 맞게 2줄 셀로 묶는다.
   - 상태는 텍스트 색상만 쓰지 말고 `status ok/wait/fail/info` chip으로 표현한다.
   - 행 클릭 시 우측 상세 drawer가 열린다.
   - 선택된 행은 강조 표시한다.

2. 검색/필터
   - 제목, 담당자, 댓글, 세부내용 통합 검색을 제공한다.
   - 대분류/상태/우선순위/마감임박 필터를 제공한다.
   - 선택된 필터는 active filter tag로 요약한다.
   - Summary Strip에는 전체 업무, 진행중, 지연주의, 완료, 첨부 있음 건수를 조밀하게 표시한다.

3. 신규 업무 생성
   - `index2.html`의 가로 압축형 생성 폼을 참고한다.
   - 필수 필드:
     - 시작일
     - 대분류/중분류/소분류
     - 업무 제목
     - main/HOST 마감일
     - main 담당자
     - sub 협업자 다중 선택
     - 세부내용 1~3번 가이드
     - 중요 및 특이사항
     - 공유자료명
     - 상태/우선순위
   - 등록 시 업무 목록에 새 행을 추가하고, 해당 행을 선택 상태로 만들며 우측 상세를 즉시 갱신한다.

4. 업무 상세
   - `../notations_tools/public/index.html`의 상세 구조를 참고한다.
   - `테이블선택_Drawer_상세패널_기준.md`를 함께 적용해, 단순 상세 보기 패널이 아니라 선택된 업무를 처리하는 작업 패널로 만든다.
   - 상세는 다음 영역으로 구성한다.
     - 제목/경로/상태/우선순위
     - 업무 메타 정보
     - 세부내용
     - 중요 및 리스크
     - 첨부파일 목록 및 미리보기/다운로드 버튼
     - 활동 히스토리
     - 댓글 및 대댓글
     - 운영 메모
     - 상태 변경/저장 액션
   - 댓글 입력 시 타임라인에 즉시 추가된다.
   - 댓글/변경/파일 활동을 탭으로 필터링할 수 있게 한다.
   - 상세 drawer에는 닫기, 넓게보기, 저장 액션을 둔다.
   - 테이블 행은 클릭뿐 아니라 Enter/Space 키로도 선택 가능하게 한다.
   - 선택된 행에는 `tr.is-open` 또는 동일 의미의 선택 상태 클래스를 적용한다.
   - Drawer가 열리면 작업영역에는 `is-drawer-open`, 넓게보기에는 `is-drawer-wide` 상태를 적용한다.
   - 닫기 또는 Escape 동작 시 선택 행 상태와 wide 상태를 복구하고, 포커스를 선택했던 행으로 돌린다.
   - Drawer 내부 정보는 고정 정보와 변동 정보를 분리한다.
     - 고정 정보: 업무번호, 제목, 생성인, 대/중/소 분류, main 담당자, sub 협업자, 마감일
     - 변동 정보: 상태, 우선순위, 댓글, 활동 이력, 첨부, 운영 메모, 리스크 처리 상태
   - 상세 패널은 업무 의미별 section으로 나누고, 각 값은 라벨 + field 구조로 표시한다.
   - 누적되는 댓글, 활동 히스토리, 파일 이력은 하나의 textarea에 합치지 않고 별도 리스트로 유지한다.

디자인 기준:
- BO 관리자 화면답게 고밀도, 저여백, 실무형 UI로 만든다.
- 과한 hero, 마케팅 카드, 장식 이미지는 금지한다.
- 기본 폰트는 Pretendard 계열, BO 테이블은 8~11px 밀도를 기준으로 한다.
- Topbar 높이는 약 48px, Sidebar는 약 224px 기준으로 한다.
- 테이블 헤더는 밝은 회색, 셀 경계는 얇은 라인으로 구성한다.
- radius는 4~6px 수준으로 제한한다.
- chip/status는 연한 배경 + border + 텍스트 + 상태 점을 함께 사용한다.
- 카드 안에 카드를 중첩하지 않는다.
- 긴 텍스트는 ellipsis 또는 2줄 clamp 처리한다.
- 모바일에서도 테이블을 완전히 카드화하지 말고 가로 스크롤을 허용한다.

상호작용:
- 행 클릭: 상세 drawer 데이터 바인딩
- 상세 닫기: 목록 영역 확장
- 상세 다시 열기 버튼 제공
- 신규 업무 등록: 목록 행 추가 + 상세 갱신
- 검색/필터: 테이블 행 실시간 필터링
- 댓글 등록: 상세 댓글 목록과 활동 히스토리에 반영
- 첨부 미리보기: 간단한 preview modal 또는 inline preview 영역 표시

산출물:
- 단일 실행 가능한 HTML/CSS/JS 프로토타입
- 파일명은 `work_communication_tool.html`로 생성
- 외부 서버 없이 브라우저에서 바로 열 수 있어야 한다.
- 샘플 데이터는 최소 5건 이상 포함한다.
- 코드 안에 불필요한 설명 문구를 화면에 노출하지 않는다.
```

## 구현 시 주의사항

- `index2.html`, `index.html`, `notations_tools`의 스타일은 복사하지 않는다. 해당 파일은 필요한 데이터 필드와 동작 흐름만 참고한다.
- 상세 화면의 시각 구조와 간격, 라인, 상태 표현은 디자인 시스템 문서의 Drawer 기준을 따른다.
- 상세 Drawer는 `테이블선택_Drawer_상세패널_기준.md`처럼 선택 행의 고정 식별 정보와 실제 처리 입력 영역을 분리한다.
- Drawer 내부 grid는 wrapper 또는 module 컨테이너가 담당한다. row 클래스에 임의의 `grid-template`, `border-bottom`, 배경 구분선을 추가하지 않는다.
- 목록에서는 조밀하게 보여주고, 상세 drawer에서는 편집 가능한 단위로 풀어준다.
- overlay modal은 첨부 미리보기 정도에만 제한하고, 업무 상세는 반드시 inline drawer로 처리한다.
- 날짜 표기는 BO 기준에 맞게 `YY-MM-DD`를 기본으로 한다.
- 상태/우선순위는 색상 텍스트만 쓰지 않고 chip 또는 badge로 처리한다.
