README만 다운로드

    핵심 사용 방식

    전역 문서

    agent.md, rules.md, prompt.md는 모든 작업에 적용되는 기본 역할과 규칙입니다.

    로컬 문서

    sites/, packages/ 아래 문서는 해당 폴더의 차이점, 상태, 설계만 기록합니다.

    권장 읽기 순서

    1. rules.md → 2. agent.md → 3. docs/design.md → 4. local/index.md → 5. local/rules.md → 6. local/context.md

    루트 문서 역할

    파일 역할 언제 읽나 관리 원칙
    README.md 전체 구조와 핵심 원칙을 설명하는 시작 문서 처음 구조를 파악할 때 전체 개요만 간결하게 유지
    index.md 작업 유형별로 어떤 문서를 읽을지 안내하는 문서 라우터 작업 시작 시 짧고 명확한 읽기 순서 유지
    agent.md 에이전트의 역할, 책임, 금지사항, 의사결정 원칙 정의 항상 정체성과 행동 원칙만 유지
    prompt.md 작업 실행 절차, 질문 기준, 보고 형식 정의 작업 수행 방식이 필요할 때 실행 절차 중심으로 관리
    rules.md 전역 규칙의 단일 진실 공급원 항상 보안, 데이터, Git 안전 규칙은 로컬에서 덮어쓰지 않음
    skills.md 상황별 도구와 스킬 선택 기준 정의 작업 방식이나 도구 선택이 필요할 때 도구 목록보다 선택 기준을 우선
    context.md 워크스페이스의 현재 목표, 제약, 알려진 이슈 정리 현재 상태 파악 시 오래된 정보는 제거하고 최신 상태만 유지
    todo.md 전체 작업 큐와 완료 기준 관리 진행 상황 확인 시 실행 가능한 체크리스트로 유지
    error.md 반복 오류, 원인, 해결책, 재발 방지 기록 오류 해결 또는 재발 방지 시 재발 가능한 오류만 기록
    handoff.md 세션 인계, 변경 파일, 다음 작업, 열린 질문 기록 세션 재개 또는 인계 시 다음 작업자가 바로 이어서 볼 수 있게 작성

    설계 문서 역할

    파일 역할 핵심 내용
    docs/design.md Docs 전체 아키텍처와 상속 우선순위 설명 Plan A + Plan B 융합 구조, 충돌 해결 기준
    docs/decision.md ADR 중요한 설계 결정 기록 결정 배경, 선택지, 채택 이유, 결과
    docs/glossary.md 용어 공통 용어 사전 Root, Local, SSOT, Override, Context, Handoff

    로컬 문서 역할

    위치 역할 사용 방식
    sites/brand-a/index.md Local Brand A 폴더의 로컬 문서 라우터 이 폴더에서 작업할 때 먼저 읽음
    sites/brand-a/rules.md Local 브랜드별 허용 오버라이드와 추가 규칙 전역 규칙을 복사하지 않고 차이점만 기록
    sites/brand-a/context.md Local Brand A의 현재 목표, 상태, 제약 현재 작업 맥락 확인
    sites/brand-a/design.md Local Brand A의 UI와 콘텐츠 설계 기준 브랜드별 화면 또는 카피 작업 시 확인
    sites/brand-a/todo.md Local Brand A의 작업 목록 폴더 단위 진행 상황 관리
    sites/brand-a/error.md Local Brand A에서 반복되는 오류 기록 동일한 실수 방지
    packages/ui/index.md Local UI 패키지 문서 라우터 공통 컴포넌트 작업 시 확인
    packages/api/index.md Local API 패키지 문서 라우터 API, 인증, 데이터 변경 작업 시 확인

    상속 구조

    계층 문서 담당 역할 오버라이드 가능 여부
    Root rules.md 보안, 데이터, 파일 수정, Git 안전 규칙 불가 또는 제한적
    Root agent.md, prompt.md 에이전트 역할과 작업 절차 일반적으로 불가
    Docs docs/design.md 전체 설계 원칙과 충돌 해결 기준 의사결정 기록 후 변경
    Local sites/*/rules.md, packages/*/rules.md 도메인별 차이점과 허용된 오버라이드 브랜드 톤, UI 밀도, 네이밍 등 가능
