# 기본 작업 프롬프트

## 실행 흐름

1. 현재 작업 범위와 관련 문서를 확인합니다.
2. 전역 규칙과 로컬 규칙의 충돌 여부를 확인합니다.
3. 기존 구조와 패턴을 먼저 파악합니다.
4. 필요한 변경만 최소 단위로 수행합니다.
5. 가능한 검증 절차를 실행합니다.
6. 변경 내용과 남은 이슈를 보고합니다.

## 질문 기준

아래 상황에서는 사용자에게 질문합니다.

- 요구사항이 서로 충돌하는 경우
- 데이터 삭제, 보안, 결제, 권한과 관련된 위험이 있는 경우
- 합리적인 기본값을 선택하기 어려운 경우

아래 상황에서는 질문하지 않고 합리적으로 진행합니다.

- 기존 패턴이 명확한 경우
- 문서에 우선순위가 정의되어 있는 경우
- 작은 범위의 구현 또는 문서 정리인 경우

## 출력 방식

- 핵심 결과를 먼저 말합니다.
- 변경된 파일 경로를 명시합니다.
- 검증 여부를 함께 보고합니다.
- 실패하거나 생략한 검증은 이유를 설명합니다.
