# 스킬 사용 지침

## 목적

이 문서는 AI 에이전트가 사용할 수 있는 도구와 스킬을 언제 선택해야 하는지 정의합니다.

## 기본 원칙

- 작업에 필요한 최소 스킬만 사용합니다.
- 공식 문서나 로컬 문서가 있는 경우 추측보다 문서를 우선합니다.
- 코드 수정 전에는 관련 구조와 기존 패턴을 먼저 확인합니다.

## 스킬 선택 기준

| 상황 | 우선 스킬 또는 접근 |
|---|---|
| 코드 리뷰 | 리뷰 규칙, 테스트 결과, 변경 diff 확인 |
| UI 구현 | 디자인 문서, 로컬 UI 규칙, 기존 컴포넌트 확인 |
| API 구현 | API 설계 문서, 데이터 모델, 인증 규칙 확인 |
| 오류 해결 | `error.md`, 로그, 재현 절차 확인 |
| 새 프로젝트 구조화 | `index.md`, `rules.md`, `docs/design.md` 확인 |

## 금지 사항

- 스킬 이름만 보고 무조건 사용하지 않습니다.
- 사용자가 요구하지 않은 대규모 리팩터링을 수행하지 않습니다.
- 검증 없이 완료로 보고하지 않습니다.
