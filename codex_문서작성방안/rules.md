# 전역 규칙

## 상속

이 파일은 워크스페이스 전역 규칙의 단일 진실 공급원입니다.
하위 폴더의 규칙은 이 파일을 확장하거나, 명시적으로 허용된 범위에서만 오버라이드할 수 있습니다.

## 우선순위

1. 보안 규칙
2. 데이터 무결성 규칙
3. 제품/도메인 규칙
4. 로컬 폴더 규칙
5. 스타일 선호사항

## 코딩 규칙

- 기존 프레임워크와 네이밍 규칙을 따릅니다.
- 명확한 이유 없이 새 의존성을 추가하지 않습니다.
- 동작이 바뀌는 경우 테스트 또는 검증 절차를 추가합니다.

## 파일 수정 규칙

- 요청된 작업 범위 안에서만 파일을 수정합니다.
- 생성 파일, 벤더 파일, 외부 라이브러리 파일은 임의로 수정하지 않습니다.
- 작업 상태가 변경되면 관련 `todo.md`를 업데이트합니다.

## 보안 규칙

- 비밀키, 토큰, 인증 정보는 문서나 코드에 직접 기록하지 않습니다.
- 사용자 데이터 삭제 또는 마이그레이션은 명시적 승인 없이 수행하지 않습니다.
- 권한, 인증, 결제 관련 변경은 별도 검증 절차를 기록합니다.

## 오버라이드 정책

로컬 `rules.md`가 오버라이드할 수 있는 항목:

- 네이밍 관례
- UI 레이아웃 관례
- 도메인별 검증 규칙
- 브랜드 톤앤매너

로컬 `rules.md`가 오버라이드할 수 없는 항목:

- 보안 규칙
- 데이터 삭제 규칙
- 비밀키 처리 규칙
- Git 안전 규칙
