# 타임스 TESOL 사이트 통합 방향 제안

작성일: 2026-05-22  
대상 사이트:

- https://timestesol.com/
- https://timestesol.imweb.me/
- https://hu-tec.github.io/TESOL-v3/courses/tesol

## 1. 통합 원칙

최종 사이트는 `TESOL-v3`를 구현 베이스로 삼는 것이 가장 적합하다. 이유는 라우팅, 반응형 UI, 신청서, 과정 상세 랜딩, 커뮤니티 구조가 이미 갖춰져 있기 때문이다.

다만 콘텐츠 원천은 `timestesol.com`에서 가져와야 한다. 오래된 사이트지만 실제 공지사항, 입학안내, FAQ, Q&A, 졸업생 후기, 연혁이 있는 운영 자산이다.

브랜드 메시지는 `timestesol.imweb.me`에서 선별해 가져오는 것이 좋다. AI 시대, 글로벌 커뮤니케이션, 사람 중심 교육이라는 메시지는 최신 교육 포트폴리오를 설명하는 데 유리하다.

정리하면 다음 역할 분담이 적합하다.

| 사이트 | 통합 역할 |
|---|---|
| timestesol.com | 실제 운영 콘텐츠와 신뢰 자료 원천 |
| timestesol.imweb.me | 최신 브랜드 메시지와 과정 포트폴리오 참고 |
| TESOL-v3 | 최종 사용자 경험과 기술 구현 베이스 |

## 2. 최종 정보 구조 제안

TESOL 중심성을 유지하면서 AI/ITT 과정은 확장 과정으로 배치한다.

| 1차 메뉴 | 하위 구조 | 비고 |
|---|---|---|
| 홈 | 핵심 메시지, 과정 카드, 성과 지표, 상담 CTA | TESOL을 첫 번째 주력 상품으로 노출 |
| 교육원 소개 | 브랜드 소개, 연혁, 협약/네트워크, 오시는 길 | 기존 사이트의 신뢰 자산 이관 |
| TESOL 과정 | 과정 소개, 커리큘럼, 수료/자격, 진로, FAQ, 신청 CTA | 최우선 상세 페이지 |
| AI 과정 | AI 번역·통역, AI 프롬프트, AI 윤리 | Imweb 메시지 활용 |
| ITT 통번역 | 정통 통번역, 전문/비즈니스 통번역 | TESOL과 분리해 혼선 방지 |
| 입학안내 | 전형 절차, 응시자격, 수업료, 학사일정, 제출서류 | 전환 전 핵심 정보 |
| 커뮤니티 | 공지사항, 보도자료, FAQ, Q&A, 졸업생 후기 | 기존 사이트 데이터 이관 |
| 문의/신청 | 레벨테스트, 설명회, 수강신청, 자격증 신청, 1:1문의 | 신청서 시스템으로 통합 |

제외 또는 분리:

- `i-TIMES`: 외부 링크 또는 별도 서비스로 Footer/브랜드 패밀리에 배치
- `출강`: B2B/기업교육 메뉴로 별도 운영
- `통역/번역`: 교육 과정이 아니라 서비스라면 별도 서비스 사이트 또는 Footer 링크로 분리

## 3. 콘텐츠 통합 방안

### 3.1 홈

추천 메시지 구조:

1. Hero: `글로벌 시대, 영어로 가르치는 전문가를 양성합니다`
2. 보조 문구: `23년 교육 노하우와 글로벌 네트워크로 TESOL·AI·ITT 전문 역량을 완성합니다`
3. CTA: `TESOL 과정 보기`, `레벨테스트 신청`, `입학설명회 신청`
4. 성과 지표: 교육경험, 졸업생, 네트워크, 전문과정
5. 과정 카드: TESOL, AI 번역·통역, AI 프롬프트, AI 윤리, ITT
6. 최신 공지/학사일정/후기

주의: TESOL이 주력이라면 Hero CTA와 첫 과정 카드는 반드시 TESOL 중심이어야 한다.

### 3.2 TESOL 과정

TESOL-v3의 `/courses/tesol` 구조를 유지하되, 기존 사이트의 신뢰 표현을 보강한다.

반드시 들어갈 항목:

- 8주 완성
- 120시간 몰입 과정
- 100% 영어 수업
- 국제 TESOL 커리큘럼
- 수업 시연/발표 실습
- 수료증/자격증 발급 기준
- 진로: 국내 강사, 방과후 영어지도, 글로벌 교육 커리어, 비즈니스 활용
- FAQ와 수강 후기

검증 필요 항목:

- `국제 공인`, `세계 어디서나 통하는`, `국내 유일` 등 법적/광고 심의 리스크가 있는 표현
- 협약 대학명, 자격증 발급 기관명, 공증/인정 관련 표현

### 3.3 커뮤니티

기존 `timestesol.com`의 커뮤니티를 새 사이트로 이관한다.

이관 우선순위:

1. 공지사항
2. 학사일정
3. FAQ
4. Q&A
5. 졸업생 후기
6. 보도자료
7. 뉴스레터

새 구조에서는 과정 필터를 추가하는 것이 좋다.

- 전체
- TESOL
- AI 과정
- ITT
- 입학/학사
- 자격증

## 4. 디자인/브랜드 방향

추천 방향:

- 브랜드명: `A. TESOL by Times Media` 또는 `타임스 TESOL`
- 톤: 신뢰, 전문성, 글로벌, 실무 중심
- 색상: TESOL-v3의 와인/딥 네이비 계열 유지
- Imweb의 AI 메시지는 보조 섹션에 사용
- 구형 사이트의 레거시 감성은 배제하고, 연혁/자료만 가져온다.

주의할 점:

- AI 메시지가 TESOL의 핵심 가치를 덮지 않게 한다.
- TESOL, ITT, AI가 모두 같은 비중이면 사용자가 무엇을 신청해야 하는지 모호해진다.
- `타임스미디어`, `타임스테솔`, `A. TESOL`의 관계를 첫 화면 또는 About에 명확히 설명해야 한다.

## 5. 기술 통합 방향

TESOL-v3 기준으로 다음 보강이 필요하다.

| 영역 | 작업 |
|---|---|
| SEO | `/courses/tesol` 등 주요 페이지별 title, description, OG 태그, 정적 HTML 또는 prerender 적용 |
| 라우팅 | Footer의 `#` 링크를 `/apply/level-test`, `/apply/seminar`, `/contact` 등 실제 경로로 연결 |
| 데이터 | 공지사항/FAQ/후기 데이터를 JSON, CMS, DB 중 하나로 구조화 |
| 신청서 | 신청 API 운영 환경 확인, 에러 로그/관리자 조회 화면 연결 |
| 연락처 | 전화번호, 이메일, 주소 기준값 통일 |
| 접근성 | 이미지 대체 텍스트, 폼 label, 키보드 이동 검토 |
| 콘텐츠 검증 | 법적 표현, 협약/자격증/수료증 문구 증빙 확인 |

## 6. 단계별 실행 계획

### 1단계: 기준 정보 확정

- 최종 브랜드명 결정
- 대표 전화번호/이메일/주소 확정
- TESOL 자격증/수료증 발급 문구 확정
- 현재 모집 과정, 수업료, 일정 확정

### 2단계: 콘텐츠 이관

- `timestesol.com` 커뮤니티 데이터 정리
- 공지/FAQ/후기 카테고리 매핑
- 불필요한 ITT/출강/외부 서비스 글 분리
- 오래된 게시글은 아카이브 처리

### 3단계: TESOL-v3 반영

- 홈 문구 정리
- TESOL 상세 페이지 실제 데이터 반영
- 입학안내 실제 일정/수업료 반영
- 신청서 링크와 Footer 링크 수정
- 커뮤니티 실제 데이터 연결

### 4단계: 공개 전 검수

- 모바일/데스크톱 화면 검수
- 신청서 제출 테스트
- SEO 메타 검수
- 오탈자 검수
- 법적 표현 검수
- 기존 도메인 리다이렉트 계획 수립

## 7. 최종 권장안

최종 통합은 `TESOL-v3`를 메인 코드베이스로 확정하고, `timestesol.com`의 운영 콘텐츠를 이관하며, `imweb.me`의 브랜드 메시지를 선별 반영하는 방향이 가장 효율적이다.

메인 도메인은 최종적으로 `timestesol.com`을 유지하는 것이 좋다. 기존 도메인의 검색 이력과 브랜드 인지도를 살릴 수 있기 때문이다. Imweb 주소는 폐기하거나 내부 시안 보관용으로 두고, GitHub Pages 주소는 개발/스테이징 용도로만 사용하는 구성이 적합하다.
