Codex 자동 승인 모드 방법

승인 질문을 줄이면서 파일 쓰기 범위는 현재 프로젝트 폴더로 제한하는 실행 방식을 한 화면에서 확인합니다.

원문 Markdown 열기 Markdown 다운로드

추천 실행값

현재 폴더 안에서만 쓰기 가능하게 두고 승인 질문을 최소화하려면 프로젝트 루트에서 실행합니다.

codex -s workspace-write -a never

권한 의미

workspace-write

작업 폴더 안의 파일 수정과 일반 로컬 명령 실행을 허용합니다. 폴더 밖 쓰기는 제한됩니다.

never

승인 프롬프트를 띄우지 않고 정해진 샌드박스 안에서 가능한 작업만 진행합니다.

주의점

현재 폴더 안의 파일 변경이나 삭제는 가능하므로 Git 커밋 또는 백업 후 사용하는 편이 안전합니다.

자동 승인에 적합한 명령

npm run test npm run lint npm run build pytest cargo test

공식 Codex 매뉴얼에서도 샌드박스와 승인 정책은 별도 제어로 설명됩니다. 자동 승인은 전체 권한 해제가 아니라, 작업 경계 안에서 멈춤을 줄이는 설정으로 다루는 것이 안전합니다.

자동 승인에서 제외할 명령

rm git reset git clean sudo chmod npm install

삭제, 되돌리기, 권한 변경, 설치, 배포, 네트워크 작업은 반복 작업이라도 수동 확인을 남기는 편이 안전합니다.
