## 2. 분석자료 확인 방법

분석자료는 일반 산출물과 분리해서 [analyze.html](./analyze.html)에서 먼저 확인합니다. 이 화면은 사이트 분석, FAQ 분석, 내부 플랫폼 분석, UI 개선 분석, 정보구조/사이트맵 분석을 한 곳에서 검색하고 미리볼 수 있게 만든 전용 화면입니다.

| 구분 | 위치 / 링크 | 먼저 확인할 때 |
| --- | --- | --- |
| 분석자료 통합 화면 | [analyze.html](./analyze.html) | 분석 문서를 제목, 경로, 분류로 검색할 때 |
| 분석 ZIP | [downloads/분석.zip](./downloads/분석.zip) | 분석자료만 묶어서 전달하거나 백업할 때 |
| 타임즈 테솔 분석 | [타임즈_테솔_분석자료](./타임즈_테솔_분석자료/) | 교육/테솔 사이트 방향을 확인할 때 |
| FAQ 분석 | [테솔_ITT_FAQ분석](./테솔_ITT_FAQ분석/) | FAQ, 예상 질문, 고객 응대 문구를 정리할 때 |
| 내부 플랫폼 분석 | [내부 관리망분석/업무플랫폼 분석](./내부 관리망분석/업무플랫폼 분석/) | 휴텍씨 사내 Studio, 인트라넷, Work Studio 통합 방향을 볼 때 |
| UI 개선 분석 | [내부 관리망분석/사이드바_개선안](./내부 관리망분석/사이드바_개선안/) | BO 사이드바 구조와 디자인 개선안을 볼 때 |
| 시험 홈페이지 분석 | [휴텍씨_시험_평가_사이트/시험_홈페이지_분석](./휴텍씨_시험_평가_사이트/시험_홈페이지_분석/) | 시험 플랫폼의 Figma/PDF/AI 분석 결과를 볼 때 |
| ITT 홈페이지 분석 | [ITT_홈페이지 분석안](./ITT_홈페이지 분석안/) | ITT-AI 시험 홈페이지 통합 방향을 확인할 때 |

휴텍씨 홈페이지는 `교육`, `시험`, `협회` 3개 랜딩페이지 구조를 기준으로 구성합니다.

| 영역 | 기준 자료 / 사이트 | 설명 |
| --- | --- | --- |
| 시험 | [타임즈 테솔](https://timestesol.com/), [AITE](https://aite-test.imweb.me), [ITT](https://itt.or.kr/web/itt/intro.php) | 기존 ITT 소개를 기반으로 그리너스, AITE 자료를 통합해 미래사업 메시지를 강조 |
| 교육 | [TESOL-v3 과정](https://hutec.github.io/TESOL-v3/courses/tesol), [AI Teacher](https://hu-tec.github.io/ai-teacher) | 교육 페이지 안에 강사 영역을 하위 탭으로 추가하는 구조 |
| 협회 | [ITT 협회 소개](https://itt.or.kr/web/itt/intro.php) | 협회 소개, 자격관리, 연혁, 윤리 기준 성격의 콘텐츠 구성 |

## 4. 디자인시스템

휴텍씨 화면 제작은 디자인시스템을 기준으로 맞춥니다.

| 파일 / 폴더 | 용도 |
| --- | --- |
| [디자인시스템/index.html](./디자인시스템/index.html) | 디자인시스템 메인 인덱스 |
| [디자인시스템/bo-index-sample.html](./디자인시스템/bo-index-sample.html) | BO 화면의 기본 구조 샘플 |
| [디자인시스템/docs_rendered/design-system.html](./디자인시스템/docs_rendered/design-system.html) | 렌더링된 디자인시스템 문서 |
| [디자인시스템/docs/재사용_디자인_프롬프트/00_문서목록.md](./디자인시스템/docs/재사용_디자인_프롬프트/00_문서목록.md) | 관리자/사용자 화면 제작 기준 문서 목록 |

중요 기준:

- 3-4단 그리드 구조를 우선합니다.
- 대표님은 여백이 많지 않은 고밀도 구조를 선호합니다.
- BO 테이블은 한 셀에 2줄 구조로 많은 정보를 담는 방향이 좋습니다.
- 회원 상세, 업무 상세 등은 우측 상세 패널로 열고, 좌측 목록은 좁혀지는 구조가 적합합니다.

버전별 참고:

- 2024 자료: 전체 유저 플로우 확인에 유리합니다.
